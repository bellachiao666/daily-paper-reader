## 摘要
记忆已成为自进化智能体的标准基础，但保留经验并不等同于学习如何通过经验进化。现有的记忆智能体可以存储轨迹、检索反思或积累技能，但往往缺乏选择有用经验、据此行动、编写可复用知识以及维护不断增长的存储库的整体能力。我们提出OPD-Evolver，一个通过同策略自蒸馏培养此类智能体进化器的慢-快协同进化框架。在快循环中，OPD-Evolver与四级记忆层次结构交互，以读取、使用、编写和维护经验，实现快速的测试时进化。在慢循环中，结果校准的记忆归因和特权后见将这四个能力蒸馏到可部署的策略中。在多领域基准测试中，OPD-Evolver比ReasoningBank等记忆系统高出最多11.5%，比Skill0等基于训练的方法高出约5.8%。进一步分析表明，OPD-Evolver内化了高价值经验和记忆管理，使得OPD-Evolver-9B能够挑战Qwen3.5-397B-A17B和Step-3.5-Flash等巨型对手，超越了记忆增强型智能体，走向真正合格的智能体进化器。

## Abstract
Memory has become a standard substrate for self-evolving agents, yet retaining experience is not the same as learning how to evolve through it. Existing memory agents can store trajectories, retrieve reflections, or accumulate skills, but often lack the holistic competence to select useful experience, act on it, write reusable knowledge, and maintain a growing repository. We introduce OPD-Evolver, a slow-fast co-evolution framework that cultivates such an agent evolver through on-policy self-distillation. In the fast loop, OPD-Evolver interacts with a four-level memory hierarchy to read, use, write, and maintain experience for rapid test-time evolution. In the slow loop, outcome-calibrated memory attribution and privileged hindsight distill these four abilities into the deployable policy. Across multi-domain benchmarks, OPD-Evolver surpasses memory systems such as ReasoningBank by up to 11.5%, and training-based methods such as Skill0 by ~5.8%. Further analysis shows that OPD-Evolver internalizes high-value experience and memory management, enabling OPD-Evolver-9B to challenge giant counterparts such as Qwen3.5-397B-A17B and Step-3.5-Flash, pointing beyond memory-augmented agents toward genuinely qualified agent evolvers.

## 2. 论文提出的方法论：核心思想、关键技术细节
- **核心思想**：定义一个合格的智能体进化器应具备四种耦合能力：① 经验选择（从噪声存储中识别有用记忆）；② 经验驱动的执行（将选中的经验转化为有效多步行动）；③ 经验编写（从新轨迹和反馈中提取可复用知识）；④ 经验管理（对记忆进行评分、整合、更新和淘汰）。OPD-Evolver 通过“快循环 + 慢循环”实现这些能力的联合训练。
- **关键技术细节**：
  - **四级记忆层次（Hierarchical Experience Memory）**：包括轨迹（trajectory）、技巧（tip）、技能（skill）和工具（tool）四类记忆，按不同粒度存储，适用于不同复用场景。
  - **快循环（Fast Loop）**：在线环境下，智能体根据任务检索候选记忆，通过选择器压缩为上下文，执行任务；之后编写新记忆并定期进行存储库维护（如合并、删除）。
  - **慢循环（Slow Loop）**：利用任务结果（标量奖励 Rt）进行**结果校准归因（Outcome-calibrated Attribution）**，为每条候选记忆计算价值分数 V(m)；然后基于特权后见的**同策略自蒸馏（On-Policy Self-distillation）**，对选择、执行、编写和维护四个子任务分别构建学生-教师对，最小化学生分布与教师分布之间的KL散度，将教师的后见知识蒸馏到可部署的学生策略中。
  - **优化目标**：统一慢循环损失 L_slow(θ) = Σ 各决策类的期望下逐token蒸馏损失，最终得到可在测试时独立运行的学生策略。
- **算法流程**（文字说明）：
  1. 初始化记忆库 M0 和策略 πθ。
  2. 对每个任务 xt：形成查询向量 → 从各记忆层检索候选 → 选择器压缩得到上下文 ct → 智能体执行获得轨迹和奖励 → 编写新记忆并更新存储库；每 Q 个任务后执行维护操作。
  3. 记录所有交互日志，计算每段记忆的结果校准归因分数。
  4. 对四个决策子任务（选择、执行、编写、维护），构造学生输入 zk 和教师额外后见输入 hk，学生生成输出，教师基于相同前缀生成分布，计算KL散度作为蒸馏损失。
  5. 更新策略参数，仅部署学生侧行为。
