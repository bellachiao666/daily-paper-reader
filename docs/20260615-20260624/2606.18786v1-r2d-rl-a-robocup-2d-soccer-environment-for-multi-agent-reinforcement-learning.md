## 摘要
机器人足球是多智能体强化学习的一个具有挑战性的测试平台，因为它结合了部分可观测性、合作与对抗交互、稀疏奖励以及长时域战术行为。RoboCup 2D足球仿真（RCSS2D）提供了一个成熟的机器人足球平台，但其面向竞争的服务器-客户端架构难以直接用于基于Python的现代多智能体强化学习工作流。我们提出了R2D-RL，一个通过共享内存通信和周期级同步将RCSS2D和基于HELIOS的玩家客户端连接到Python多智能体强化学习接口的强化学习环境。R2D-RL支持全场比赛和基于场景的训练，具有可配置的对手、基础离散和混合参数化动作空间、动作掩码、基于期望控球值（EPV）的奖励塑造以及并行执行。我们提供了前场射门场景和11对11全场比赛基准，以及基线结果。

## Abstract
Robot soccer is a challenging testbed for multi-agent reinforcement learning because it combines partial observability, cooperative and adversarial interaction, sparse rewards, and long-horizon tactical behavior. RoboCup 2D Soccer Simulation (RCSS2D) provides a mature robot-soccer platform, but its competition-oriented server-client architecture is difficult to use directly with modern Python-based MARL workflows. We introduce R2D-RL, a reinforcement learning environment that connects RCSS2D and HELIOS-based player clients to a Python MARL interface through shared-memory communication and cycle-level synchronization. R2D-RL supports full-field and scenario-based training with configurable opponents, Base discrete and Hybrid parameterized action spaces, action masks, expected possession value (EPV)-based reward shaping, and parallel execution. We provide front-goal scenarios and an 11-vs-11 full-field benchmark, together with baseline results.

## 2. 方法论
- **核心思想**：通过共享内存通信和周期级同步，将原始 RCSS2D 服务器、修改后的 HELIOS 客户端（玩家、教练、训练器）与 Python MARL 环境连接，把竞赛流程转变为步进同步的强化学习交互，暴露训练所需的观测、动作、奖励、终止信号、动作掩码和复位控制。
- **关键技术细节**：
  - **步进同步通信协议**：每个受控玩家维护 `obs_seq`（观测发布）、`act_seq`（动作分配）、`done_seq`（动作消费）三个计数器，外加 `phase`（比赛模式）和 `cycle`（周期）。仅在 `play_on` 模式下暴露策略决策；一个环境步对应一个同步的 `play_on` 决策周期。

    - 局部观测（玩家侧）：97 维向量（自身 6 维 + 球 4 维 + 11 个对手 × 4 维 + 10 个队友 × 4 维 + 3 维指示符），不足时零填充。
  - **动作空间**：
    - **Base 离散动作空间**：19 个高语义动作（如射门、传球、盘带、移动等），由 HELIOS 行为模块负责底层执行。

  - **动作掩码**：基于 HELIOS 可执行性检查动态生成，射门为优先级动作（启用时屏蔽其他所有 Base 动作）。
  - **奖励**：团队得分奖励（+1 进球，-1 失球） + 可选的 MaxEPV 塑形奖励（基于预计算 32×50 EPV 网格，只计算持球方接近球时的增量，进球时额外奖励达到网格最大值）。
  - **终止条件**：进球、死球、失球（防守方更近的控球丢失）、超时。
  - **场景重置**：通过训练器直接移动球和球员位置（不重启服务器），支持内置场景和用户自定义初始状态（位置、方向、速度）。
  - **内置 AI**：对手和未控队友分配 HELIOS Fallback 动作。

## 3. 实验设计
- **场景与基准**：
  - **前场场景（5 个难度递增）**：空门（1v0）、封堵射门（1v1）、支援选项（2v2）、传球通道（3v3）、紧凑防守（3v4）。所有场景均朝右进攻，由学习策略控制进攻方，对手使用 HELIOS Fallback。
  - **全场比赛基准**：11v11 标准 RCSS2D 全场，左队由策略控制，对手为 HELIOS Fallback。超时 3000 环境步终止；进球不终止但导致得分奖励。
- **对比方法**：
  - Base 动作空间：MAPPO（多智能体 PPO）、QMIX（单调值函数分解）。

## 6. 主要结论与发现
- R2D-RL 能够有效连接 RCSS2D 与 Python MARL，支持同步、并行采样和渐进式实验。
- **并行采样**：Base 空间最高 34.85M 步/天，Hybrid 空间 41.65M 步/天（14 实例达到峰值）。
- **前场场景**：
  - 空门：所有算法基本可达接近 100% 进球率。
  - 封堵射门：QMIX 依赖动作掩码（mask off 时 0%），MAPPO 表现更鲁棒，ParaDQN 在 mask+EPV 下达到 28%。
  - 多攻击者场景：仅 MAPPO 在紧凑防守（MaxEPV+mask 下 32%）有稳定正进球率；QMIX 在支援和传球通道有正结果；ParaDQN 几乎全零。
  - **MaxEPV 和动作掩码**在复杂任务中显著有效（无两者时几乎所有多攻击者设置进球率为 0）。
- **全场比赛**：MAPPO 表现最稳定（净胜球 -10.61，EPV 提升 0.0317），QMIX 种子变异大，ParaDQN 基本无效。定性分析显示存在固定方向跑位、缺乏门前特定门将行为等局限。

## 7. 优点
- **环境设计完整**：保留了 RCSS2D 成熟的仿真和 HELIOS 行为库，同时提供标准化 MARL 接口（观测、中心状态、动作掩码、奖励、终止、复位），易与现有框架集成。
- **支持渐进式学习**：提供从单攻击者到多攻击者再到全场的逐步难度场景，便于分析能力瓶颈。
- **并行采样高效**：通过共享内存同步实现了实际规模的并行采样（千万级/天）。
- **动作掩码和 EPV 奖励**：设计合理，实验证明能显著提升学习效率（尤其对值函数方法 QMIX 和复杂多攻击者任务）。
- **开源性**：论文声称环境将开源（基于 RCSS2D 和 HELIOS），有利于社区复现和扩展。

## 8. 不足与局限
- **对手固定**：所有实验对手均为单一 HELIOS Fallback 策略，缺乏多样性；未进行自对弈或混合对手池训练，泛化能力未知。
- **训练预算有限**：3-30M 步相对于全尺寸 11v11 可能不足，作者承认更大规模、课程学习、示范等可提升性能。
- **手工设计抽象**：动作空间和动作掩码依赖 HELIOS 规则，引入了归纳偏见；未来需探索更自动化的抽象。
- **仅 play_on 模式决策**：定位球（开球、任意球、角球等）由规则和 HELIOS 处理，未纳入学习范围，限制了策略覆盖的完整战术。
- **算法局限性**：ParaDQN（Hybrid 空间）在复杂任务上几乎无效，表明参数化动作空间与集中训练结合仍需改进。
- **硬件资源未透明**：未报告 GPU 型号、数量及具体训练时长，影响可复现性。
- **实验覆盖**：前场场景中 QMIX 在某些设置下种子变异大（标准差高），部分设置（无 mask 的 QMIX）失败未深入分析原因；全场 MAPPO 和 QMIX 的 EPV 提升数值较小（0.03），实际进攻进展有限。
