## 摘要
自进化智能体通过从过去的执行中提取经验并在未来任务中重用这些经验来随时间改进。现有系统将这些经验表示为注入智能体上下文的自然语言文本，或表示为可调用工具暴露的代码。然而，这些表示之间的选择通常在设计时做出，而不是从经验本身的特征中推导出来，导致对它们之间权衡的理解不足。我们提出了第一个对照研究，在相同的经验集上隔离文本记忆和代码记忆。我们的结果表明，这两种形式在构建成本、执行效率和可迁移性方面表现出互补的权衡，以至于单一表示不足以胜任。在这些发现的指导下，我们提出了Metis，一个构建在分层双表示记忆上的自进化智能体系统。Metis将文本经验组织成执行计划、环境事实和常见陷阱，并选择性地将重复出现的计划结晶为可验证的可调用工具。这种设计结合了文本记忆的广泛适用性和代码记忆的执行效率，同时仅在通过重复使用证明合理时才产生工具生成成本。我们在AppWorld（一个具有挑战性的交互式智能体基准）上评估了Metis。结果表明，与ReAct相比，Metis将任务准确性提高了高达20.6%，同时将执行成本降低了高达22.8%。与代表性的自进化智能体系统相比，Metis在准确性、执行效率和记忆构建成本之间始终实现了更好的平衡。

## Abstract
Self-evolving agents improve over time by distilling experience from past executions and reusing it in future tasks. Existing systems represent such experience either as natural-language text injected into the agent context or as code exposed as callable tools. However, the choice between these representations is typically made at design time rather than derived from the characteristics of the experience itself, leaving the trade-offs between them poorly understood. We present the first controlled study that isolates text memory and code memory over an identical set of experiences. Our results show that the two forms exhibit complementary trade-offs in construction cost, execution efficiency, and transferability, such that neither representation alone is sufficient. Guided by these findings, we propose Metis, a self-evolving agent system built on a hierarchical dual-representation memory. Metis organizes textual experience into execution plans, environment facts, and common pitfalls, and selectively crystallizes recurring plans into validated callable tools. This design combines the broad applicability of text memory with the execution efficiency of code memory while incurring tool-generation cost only when justified by repeated reuse. We evaluate Metis on AppWorld, a challenging benchmark for interactive agents. The results show that Metis improves task accuracy by up to 20.6% over ReAct while reducing execution cost by up to 22.8%. Compared with representative self-evolving agent systems, Metis consistently achieves a better balance between accuracy, execution efficiency, and memory-construction cost.

## 2. 方法论：核心思想与技术细节

### 核心思想
- **层次双表示记忆**：以低成本、高可迁移性的文本记忆为基础，仅对**重复出现**且**稳定**的执行计划进行“晋升”，结晶为可调用的代码工具。形成“文本 → 识别重复 → 代码”的优化循环。

### 关键技术细节
1. **差异化文本反射（Differentiated Text Reflection）**
   - 将文本经验分为三类：
     - **执行计划（Plans）**：可重用子例程的模板。
     - **环境事实（Facts）**：环境属性与约束。
     - **常见陷阱（Pitfalls）**：失败模式（触发器、错误、后果）。
   - 反射器作为智能体运行，可通过渐进式披露（保留轨迹开头和结尾，中间截断）查看轨迹，并主动检查错误原因，而非仅依赖轨迹文本。
   - 内存维护：合并冗余、废弃过时条目（通过失效位标记，而非直接删除）。

2. **模式感知代码生成（Pattern-Aware Code Generation）**
   - **晋升单位**：只有**执行计划**（具备可机械执行的子例程）才有资格晋升为代码。每次计划被选中时，都会累积“重复证据”（放入缓冲区B(p)）。
   - **触发条件**：当缓冲区大小 |B(p)| ≥ 阈值 θ 时，触发代码化。
   - **代码化过程**：代码生成器接收计划正文、相关查询（来自缓冲区和历史任务），但不接收原始轨迹，以避免过拟合错误执行路径。
   - **工具准入**：生成的工具必须通过依赖闭合、编译检查及沙箱验证，才能加入代码库。

3. **记忆管理（Memory Manager）**
   - 联合检索：基于嵌入的 top-k 检索 + 轻量级 LLM 管理器的联合选择，确保文本与代码之间协同，而非独立处理。
   - 选择计划会自动使其实现的代码工具可用。

4. **反射线束（Reflection Harness）**
   - 渐进式披露：轨迹先压缩为紧凑视图（保留动作类型、名称，截断长字段），反射器可要求扩展特定步骤。
   - 依赖闭合：选择文本或代码后，自动解析并注入所有依赖的自定义函数，确保执行环境完整。
   - 工具准入守卫：编译失败后允许一次纠正，否则拒绝。

### 算法流程（文字说明）
1. 用户请求 → 记忆管理器检索文本与代码（联合选择）。
2. 智能体执行任务，产生轨迹。
3. 轨迹执行后，开始双路径反射：
   - 若选中计划的缓冲区大小≥阈值，先调用代码化管道（查询+计划，不包含轨迹）生成工具并验证。
   - 差异化文本反射基于轨迹和更新的代码库，创建/修订文本经验。
4. 反射期间，线束提供渐进式视野；代码工具需通过编译检查才被接纳。
5. 更新后的内存写入双表示记忆，供后续任务重用。

## 3. 实验设计

  - 包含9个模拟应用，457个API。
  - 任务为自然语言指令，要求智能体在状态环境中完成复杂操作。
  - 使用两个数据划分：

  - **No Memory**：基础 ReAct 智能体，无进化记忆。

  - 所有方法共享相同智能体执行器、任务顺序和沙箱接口，仅记忆形式与更新逻辑不同。

## 6. 主要结论与发现
1. **文本记忆与代码记忆互补**：文本构建成本低、迁移可靠性高但执行效率低；代码执行效率高但构建成本高、迁移时表现差（Official 分割下降 22.3 点 vs 文本下降 5.6 点）。
2. **重复门控的有效性**：基于重复的门控提高了工具质量——完整 Metis 的代码调用率远高于急切变体（56% vs 41%）。急切变体成本高但准确率低（63.2% vs 66.7%）。
3. **Metis 性能最优**：
   - 在两种分割上均获得最高准确率（Official 60.1%，Resampled 66.1%）。
   - 执行成本最低（Official 97.4K tokens，Resampled 78.5K tokens），对比 ReAct 降低 22.8%。
   - 反射成本低于 SkillX（Official 7.9M vs 9.4M，Resampled 11.8M vs 17.8M），尽管额外做了工具合成。
4. **分布位移下优势更明显**：SkillX 在分布位移下失去大部分记忆效用（实用增益从 +10.7 降到 +3.0），而 Metis 仅从 +11.3 降到 +8.3，表现出其生成工具的更好泛化性。
