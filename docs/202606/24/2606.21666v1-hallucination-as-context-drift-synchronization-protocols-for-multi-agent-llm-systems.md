## 摘要
多智能体LLM系统经常产生无法仅用模型缺陷解释的幻觉输出。这类失败的一个重要来源并非模型能力不足，而是上下文漂移：并发智能体之间内部知识状态的分歧。当智能体带着对共享世界状态的不匹配或过时表征进入协作任务时，它们的联合推理会产生表现为幻觉的矛盾。我们定义了上下文分歧分数（CDS），这是一种轻量级标量度量，用于量化跨空间、时间和任务维度的智能体对之间的知识状态差异，并提出了共享状态验证协议（SSVP），该协议允许智能体定期交换压缩状态摘要，并在联合推理前标记高分歧条件。我们使用Claude Haiku在两个领域（多智能体旅行和软件项目规划）评估了SSVP。在8个场景的受控实验中（旅行条件n=30，软件条件n=10），朴素的全广播同步使幻觉率比无同步基线高出34%（HR：0.658 vs. 0.492，p=0.0022，d=1.18），这是传播错误智能体状态的污染效应。SSVP避免了这种失败模式，同时显示出适度且一致的降低（HR：0.463，d=0.30），并且使用少58%的API调用实现了显著低于全广播的幻觉率（p=0.0005，d=1.47）。在软件领域中，污染效应未再现，所有条件均收敛到低HR（<0.2），证实其特定于一个错误的共享信念级联跨评估维度的任务。我们的结果将幻觉缓解重新定义为分布式系统问题，并将上下文同步确立为多智能体LLM设计中的一等原语。

## Abstract
Multi-agent LLM systems routinely produce hallucinated outputs that cannot be explained by model deficiencies alone. A significant class of these failures arises not from model incapacity but from context drift: the divergence of internal knowledge states between concurrent agents. When agents enter a collaborative task with mismatched or stale representations of shared world state, their joint reasoning produces contradictions that manifest as hallucination. We define the Context Divergence Score (CDS), a lightweight scalar metric quantifying knowledge-state discrepancy between agent pairs across spatial, temporal, and task dimensions, and propose the Shared State Verification Protocol (SSVP), which lets agents periodically exchange compressed state summaries and flag high-divergence conditions before joint reasoning. We evaluate SSVP across two domains (multi-agent travel and software project planning) using Claude Haiku. In controlled experiments (n=30 per condition, travel; n=10, software) across 8 scenarios, naive full-broadcast synchronization increases hallucination rate by 34% above the no-sync baseline (HR: 0.658 vs. 0.492, p=0.0022, d=1.18), a contamination effect from propagating erroneous agent states. SSVP avoids this failure mode while showing modest, consistent reduction (HR: 0.463, d=0.30) and achieves significantly lower hallucination than full-broadcast (p=0.0005, d=1.47) using 58% fewer API calls. The contamination effect does not replicate in the software domain, where all conditions converge to low HR (<0.2), confirming it is specific to tasks where one erroneous shared belief cascades across evaluation dimensions. Our results reframe hallucination mitigation as a distributed systems problem and establish context synchronization as a first-class primitive in multi-agent LLM design.

### 论文核心问题与整体含义（研究动机和背景）

- **核心问题**：多智能体LLM系统中出现的幻觉输出（hallucination）无法仅由模型缺陷解释。作者指出一个重要但未被充分研究的根源——**上下文漂移（Context Drift）**，即并发运行的智能体之间内部知识状态不一致，导致各自正确推理却产生相互矛盾的联合输出。这不是单独模型层面的问题，而是分布式系统中状态同步的问题。
- **整体含义**：该工作将幻觉缓解重新定义为**分布式系统的一致性问题**，提出把上下文同步提升为多智能体LLM设计中的一等原语（first-class primitive）。这一视角暗示，改善模型能力无法根治这类失败，而需要引入跨智能体的状态协调机制。

### 论文提出的方法论

- **核心思想**：通过轻量级标量度量实时检测智能体间上下文分歧，并在分歧超过阈值时触发选择性同步，避免盲目传播错误状态。
- **关键技术细节**：
    1. **上下文向量表示**：每个智能体`i`在时刻`t`维护一个`d`维上下文向量`c_t^i = f_θ(s_t^i ∥ h_t^i ∥ g_t^i)`，其中`s`为空间-环境状态，`h`为任务历史，`g`为目标状态。实际实现中，智能体用LLM生成结构化JSON摘要，再通过句子编码器（sentence-transformers all-MiniLM-L6-v2）映射为384维嵌入。
    2. **上下文分歧分数（CDS）**：定义为智能体对上下文向量之间的余弦距离：`CDS(i,j,t) = 1 - (c_t^i · c_t^j) / (‖c_t^i‖‖c_t^j‖)`。范围[0,2]，阈值τ=0.25。系统级分歧为所有智能体对的平均CDS。
    3. **共享状态验证协议（SSVP）**：算法流程如下：
        - 初始化上下文向量。
        - 循环：每个智能体执行本地推理步骤。
        - 当达到同步间隔或检测到任务关键事件时，每个智能体生成压缩上下文摘要σ_t^i（提示为“用不超过3句话概括你的空间上下文、任务历史和当前目标”）。
        - 广播σ_t^i给所有同伴。
        - 计算所有智能体对的CDS；若系统CDS_sys(t) > τ，暂停联合推理。
        - 高分歧对交换完整上下文（c_t^i, c_t^j），然后通过**上下文合并（ContextMerge）** 过程进行裁决（提示智能体识别矛盾并依据时间戳和信息质量判断权威来源）。
        - 重新嵌入合并后的上下文，更新状态，恢复推理。
    4. **关键属性**：模型无关、O(1)每对每步、阈值可校准。

### 论文的主要结论与发现

1. **污染效应（Contamination Effect）**：朴素全广播同步显著恶化幻觉率（旅行领域：HR 0.658 vs 无同步0.492，p=0.0022，d=1.18）。该效应源于无条件传播一个错误智能体的状态，导致所有智能体收敛于错误信念。
2. SSVP避免了污染效应，并显示出适度的幻觉降低（HR 0.463，-5.9%，d=0.30，无统计学显著性），同时比全广播减少58% API调用，且与全广播的差距高度显著（p=0.0005，d=1.47）。
3. 污染效应是领域特定的：在软件领域未重现（全广播HR=0.150，甚至略低于无同步0.188），表明它发生在“单一错误信念级联跨越语义相关评估维度”的任务中（如目的地→机场→天气→推荐）。
4. CDS的价值在于**指示何时同步**（具规范性），而非预测幻觉（无同步时CDS与HR几乎无相关r=-0.03）。

### 优点

- **创新视角**：将多智能体LLM幻觉重新概念化为分布式系统中的状态一致性问题，而非单纯模型缺陷，启发性强。
- **轻量实用**：CDS仅需O(1)每对每步的向量点积，SSVP触发开销小（O(nλ)摘要生成+O(kλ)合并），实际部署可行。
- **实验设计严谨**：故意注入多种类型的上下文漂移（空间、时间、任务），控制变量揭示因果；使用独立LLM评估并验证裁判一致性（κ=0.79）。
- **发现新现象**：识别并实证了“污染效应”，对多智能体系统设计有重要警示意义。
- **跨领域验证**：旅行和软件领域结果对比，增强了结论的泛化性，并划定了污染效应的边界条件。
- **透明开放**：承诺开源代码、提示词和评估脚本。

### 不足与局限

- **嵌入保真度限制**：384维向量可能无法完整捕捉复杂智能体状态的细微差别，可能漏掉重要但微妙的分歧。
- **ContextMerge潜在失败**：当两个冲突源均看似权威（如不同但皆可信API数据）时，合并过程可能错误裁决。当前实验中的注入偏差足够明显，避免了此情形。
- **智能体规模限制**：仅3个智能体，而多智能体系统失败率随智能体数量非线性增加（参考文献MAST指出），SSVP在更大团队中的扩展性未经测试。
- **任务覆盖有限**：仅考察结构化规划任务，未涉及开放域对话、动态环境感知等任务，这些场景的CDS动态可能不同。
- **样本量较小**：软件领域n=10，旅行领域全广播n=15，统计功效较弱，可能影响SSVP vs No-Sync差异的显著性检出。
- **消融不完整**：未探索不同阈值τ、不同同步间隔、与其他同步方案（如固定间隔同步、基于辩论的协调）的对比。
- **阈值校准局限性**：τ=0.25基于特定校准实验，其泛化到其他嵌入模型或任务领域需重新校准。
- **未考虑异步和长时间运行**：实验在同步回合制设置中进行，实际系统中异步更新和长时间运行可能累积更多漂移。
- **模型单一**：仅用Claude Haiku，不同LLM的嵌入行为或推理特性可能影响结果普适性。
