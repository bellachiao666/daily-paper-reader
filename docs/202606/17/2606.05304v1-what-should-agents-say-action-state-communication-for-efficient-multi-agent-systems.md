## 摘要
基于大语言模型的多智能体系统通常围绕角色、流水线和轮次调度进行组织，而智能体之间传递的内容往往被保留为无约束的自然语言。然而，这种自由形式的通信会迅速膨胀token使用量，消耗共享上下文窗口，并最终影响系统性能和推理成本。我们分析了两种多智能体拓扑结构中的五种常见智能体间通信策略，发现没有固定的策略是普遍最优的。相反，有效的智能体间信息始终保留下游智能体所需的以行动为中心的信息。基于此，我们提出了PACT（协议化行动-状态通信与传输），它将智能体间通信视为一个公共状态更新问题，并在每个原始智能体输出进入共享历史之前，将其投影为紧凑的行动-状态记录。在不同的多智能体拓扑结构中，PACT一致地改善了性能-成本权衡，以更少的token实现相当或更强的任务性能。这种收益扩展到生产编码框架：PACT在每解决一个任务的token数减少10%的情况下，提升了OpenHands的解决率，并且在SWE-agent上保持解决率不变的同时，将输入token减半。我们的代码公开在https://github.com/iNLP-Lab/PACT。

## Abstract
Multi-agent systems (MAS) built on large language models are typically organized around roles, pipelines, and turn schedules, while the content that agents pass to one another is often left as unconstrained natural language. However, this free-form communication can rapidly inflate token usage, consume the shared context window, and ultimately affect both system performance and inference cost. We analyze five common inter-agent communication strategies across two MAS topologies, finding that no fixed strategy is universally optimal. Instead, effective inter-agent messages consistently preserve action-centered information needed by downstream agents. Building on this, we propose the PACT (Protocolized Action-state Communication and Transmission), which treats inter-agent communication as a public state-update problem and projects each raw agent output into a compact action-state record before it enters shared history. Across different MAS topologies, PACT consistently improves the performance-cost trade-off, achieving comparable or stronger task performance with substantially fewer tokens. The gains extend to production coding harnesses: PACT lifts OpenHands' resolve rate at -10% tokens-per-resolved, and is resolve-neutral on SWE-agent while halving input tokens. Our code is publicly available at https://github.com/iNLP-Lab/PACT.

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：将智能体间通信视为公共状态更新问题，而非自由对话。每个智能体的输出在进入共享历史前，应被投影为紧凑的“行动-状态”记录，仅保留对下游智能体有用的接收者面信息。
- **关键技术——PACT（协议化行动-状态通信与传输）**：
  - **消息空间**：每条公共消息由三个字段组成：
    - **ACTION**：发送者已采取或要求的下一个行动。
    - **STATE**：支持该行动的观察、证据或环境反馈。
    - **RESULT**：行动产生的具体输出（如检索到的事实、修改后的方案）。
  - **更新规则**：传统 MAS 直接将原始输出 y_t 追加到共享历史；PACT 在追加前应用投影函数 ΠPACT，仅保留上述三个字段，排除中间推理痕迹、重复陈述等过程内容。
  - **性质**：
    - 不约束智能体的私有计算（推理、规划仍可自由进行）。
    - 与现有 MAS 拓扑正交，无需引入新智能体、改变轮次或训练压缩器。
    - 可适配不同场景：分裂证据交互中显式写出三个字段；流水线中可通过角色隐含部分行动信息。
- **技术实现**：无需训练，作为轻量级代理钩子（proxy hook）工作，在推理时实时改写消息。

## 6. 主要结论与发现

- **无固定最优通信策略**：Full Content 昂贵冗余，Conclusion Only 在交互场景失效，Artifact Only 虽好但诱导更多轮次。有效消息通常是以行动为中心的。
- **PACT 显著提升性能-成本权衡**：
  - 在交互场景：PACT 以最少 token 取得最高 F1（如 HotpotQA 上 Qwen3-8B: F1 69.9, tok 6,704 vs. 最佳基线 CoA 68.6/7,281）。
  - 在流水线：PACT 在多数数据集上准确率最高或接近最高，token 仅约为 Multi-Agent Debate 的 19%~23%。
  - 消融证实所有三个字段均必要（移除任一字段 F1 下降约 5 点）。
  - 更强模型更有效利用紧凑通信（8B→32B 平均 token 降 21%）。
- **编码框架扩展**：
  - OpenHands：解决率提升 3.60pp（19.40%→23.00%），每解决 token 减少 10.3%。
  - SWE-agent：输入 token 减少 50.4%，每解决 token 减少 47%，解决率基本持平（25.6%→24.2%）。
- **结论**：协议化、稀疏的公共通信是一种轻量、可迁移的高效机制。

## 7. 优点

- **问题选择新颖**：聚焦“应说什么”这一被忽视的设计维度，而非传统“谁来说、何时说”。
- **方法简洁高效**：PACT 无需训练，不改变智能体内部能力，与现有 MAS 架构正交，易于集成。

- **公开代码**：促进可复现性和后续研究。
