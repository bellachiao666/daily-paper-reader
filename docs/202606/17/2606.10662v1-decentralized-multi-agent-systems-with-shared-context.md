## 摘要
多智能体系统（MAS）可以通过将复杂问题分解为并行子任务，在测试时扩展大语言模型的推理能力。然而，现有的大多数MAS依赖于集中式编排，即由一个主智能体分配工作、收集输出并合并结果。随着子任务数量的增加，这一控制器会成为通信和集成的瓶颈。我们提出去中心化语言模型（DeLM），这是一种MAS框架，通过并行智能体、共享验证上下文和任务队列实现去中心化协调。智能体异步认领子任务，读取累积进度，执行局部推理，并写回紧凑的验证更新。共享上下文充当公共通信基底，使智能体能够在彼此验证的进度基础上工作，而无需通过中央控制器路由每次更新。实验上，DeLM在软件工程测试时扩展和长上下文推理方面均有提升。在SWE-bench Verified上，DeLM在Avg.@1、Pass@2和Pass@4指标上均取得最佳性能，相比最强基线提升高达10.5个百分点，同时将每个任务的成本降低约50%。在LongBench-v2多文档问答上，DeLM在四个前沿模型系列中取得最高平均准确率，相比最强基线提升高达5.7个百分点。代码可在项目网站https://yuzhenmao.github.io/DeLM/获取。

## Abstract
Multi-agent systems (MAS) can scale large language model reasoning at test time by decomposing complex problems into parallel subtasks. However, most existing MAS rely on centralized orchestration, where a main agent assigns work, collects outputs, and merges results. As the number of subtasks grows, this controller becomes a communication and integration bottleneck. We propose Decentralized Language Models (DeLM), a MAS framework that decentralizes coordination through parallel agents, a shared verified context, and a task queue. Agents asynchronously claim subtasks, read accumulated progress, perform local reasoning, and write back compact verified updates. The shared context acts as a common communication substrate, enabling agents to build on one another's verified progress without routing every update through a central controller. Empirically, DeLM improves both software-engineering test-time scaling and long-context reasoning. On SWE-bench Verified, DeLM achieves the best performance across Avg.@1, Pass@2, and Pass@4, with gains of up to 10.5 percentage points over the strongest baseline, while reducing cost per task by roughly 50%. On LongBench-v2 Multi-Doc QA, DeLM achieves the highest average accuracy across four frontier model families, improving over the strongest baseline by up to 5.7 percentage points. The code is available on our project website at https://yuzhenmao.github.io/DeLM/.

### 2. 论文提出的方法论：核心思想、关键技术细节

*   **核心思想：去中心化协调**。论文提出 **DeLM (Decentralized Language Models)** 框架，将协调从中心控制器转移到一个共享的问题状态。智能体不再是向主智能体汇报，而是异步地从一个共享的**任务队列**中领取任务，读取**共享上下文**中的已验证进展，执行局部推理，并写回紧凑的已验证更新。

*   **关键技术细节**：
    *   **三大核心组件**：
        1.  **并行智能体 (Parallel Agents)**：多个智能体独立、异步地工作。
        2.  **共享上下文 (Shared Context)**：存储紧凑、可重用的问题状态，包括已验证的事实、失败假设、中间约束等。**所有智能体都能直接读取**。
        3.  **任务队列 (Task Queue)**：存储待处理的子任务，供智能体异步认领。
    *   **验证准入机制 (Verified Admission)**：这是DeLM的核心创新。智能体的任何输出在被写回共享上下文前，都必须经过一个“验证门”。
        *   **验证内容**：将智能体的输出（压缩成 `gist`）与它的支持证据（原始上下文或推理轨迹）进行比对，确保持续性、准确性，防止幻觉和错误传播。
        *   **验证路径**：对于推理轨迹，直接验证`gist`是否忠实于原始输出；对于长文档，进行**分层验证**：首先验证`summary`是否基于原始文本，再验证`gist`是否忠实于`summary`。
    *   **层级式共享上下文 (Hierarchical Shared Context)**：为了解决信息完整性与上下文窗口限制之间的矛盾，DeLM设计了三级结构：
        1.  **`Gist` 层**：存储在共享上下文中的最紧凑条目，是智能体默认读取的“全局视图”。
        2.  **`Summary` 层**：对于长文档，`gist`指向一个更详细的、带原始引用的摘要（`Si`）。
        3.  **`Raw` 层**：最原始的文档内容。
    *   **选择性展开机制 (Selective Unfolding)**：智能体在阅读`gist`后，如有需要，可以“展开”查阅更详细的`summary`，或进一步“深度展开”查阅原始文本。此机制类似操作系统的“请求调页”，只在必要时才消耗更多上下文。
    *   **阻塞/非阻塞I/O**：通过写后发布的原子性操作，保证缓存命中率。

1.  **去中心化显著优于集中式**：DeLM通过在SWE-bench、LongBench-v2和OOLONG（与RLM结合）上取得最佳或接近最佳的性能，证明了**去中心化、基于共享状态的协调机制**在处理多智能体测试时扩展问题时，比传统的集中式协调更有效。
2.  **共享失败与约束**：DeLM允许并行智能体共享**失败的假设**（将其变为可重用的约束）和**关键约束**，避免了其他智能体重复探索死路，并确保了问题约束的全局一致性。

5.  **互补性**：DeLM的自然语言共享上下文能力与RLM等程序化推理系统是互补的。对于需要精确行级聚合的OOLONG任务，将两者结合能取得最佳效果。

### 8. 不足与局限

*   **计算开销**：准入时验证会带来**额外的计算开销**（调用LLM进行验证）和延迟。虽然总成本降低了，但在对延迟极度敏感的场景下，这可能成为新的瓶颈。
*   **依赖分解质量**：DeLM的性能高度依赖于**初始任务分解**以及**后续子任务生成**（`GENERATE MORE SUBTASKS`）的质量。过于粗或过于细的分解都会影响效果。论文承认这是未来工作。
*   **通用性局限**：尽管DeLM在SWE和LongBench上表现优异，但在**纯结构化、计数型任务**（OOLONG）上，其自然语言共享上下文机制不如RLM的代码执行方式可靠，说明其并非通用万能。
*   **未验证的假设**：
    *   论文假设一旦更新被验证并写入共享上下文，就能被所有后续智能体正确理解。不同模型或不同提示方法对`gist`的理解可能存在差异。
    *   未讨论共享上下文的**规模问题**。当问题极其庞大或迭代次数非常多时，共享上下文本身也会变得非常庞大，内部`gist`间的冲突和冗余如何解决？
*   **潜在偏差**：论文引用的基线（如Claude Code）和基础模型的API版本是固定的，随着模型迭代，结果可能会发生变化。`gist`摘要模型（DeepSeek-V4-Flash）的性能和公平性也可能是一个潜在的未知偏差。
