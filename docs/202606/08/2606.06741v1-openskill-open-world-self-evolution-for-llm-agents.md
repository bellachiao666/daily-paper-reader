## 摘要
自我进化的智能体需要在部署后不断适应，但现有方法假设了一个可用的学习循环，例如精心设计的技能、成功轨迹或验证器信号。实际的开放世界部署可能无法提供这些，仅能给出一个任务提示。在这项工作中，我们研究了开放世界自我进化，即智能体必须从零开始构建其技能和自身的验证信号，利用开放世界资源但无需目标任务监督。我们提出了OpenSkill框架，它启动了这一循环：从文档、代码仓库和网络中获取基于事实的知识和验证锚点，将它们综合为可迁移的技能，并基于自建的虚拟任务（锚定于这些锚点而非目标答案）来完善这些技能。因此，开放世界既提供了待学习的知识，也提供了独立于监督的实践环境，目标任务监督仅保留用于最终评估。在三个基准测试和两个目标智能体上，OpenSkill在满足无监督约束的同时取得了最佳自动化通过率。分析表明，其技能无需特定模型适配即可跨模型迁移，且自建的验证器尽管从未访问过真实结果，却与真实结果保持一致。

## Abstract
Self-evolving agents requires adaptation after deployment, but existing approaches assume a usable learning loop, such as curated skills, successful trajectories, or verifier signals. Real open-world deployments may provide none of these, offering only a task prompt. In this work, we study open-world self-evolution, where an agent must build both its skills and its own verification signals from scratch, using open-world resources but no target-task supervision. We propose OpenSkill, a framework that bootstraps this loop: it acquires grounded knowledge and verification anchors from documentation, repositories, and the web, synthesizes them into transferable skills, and refines those skills against self-built virtual tasks grounded in the anchors rather than in target answers. The open world thus supplies both the knowledge to be learned and a supervision-independent practice environment, with target-task supervision reserved for final evaluation. Across three benchmarks and two target agents, OpenSkill attains the best automated pass rate while satisfying the no-supervision constraint. Analysis shows its skills transfer across models without model-specific adaptation, and its self-built verifier aligns with ground-truth outcomes despite never accessing them.

## 2. 方法论：核心思想、关键技术细节

**核心思想**：OpenSkill框架将开放世界资源同时作为“知识来源”和“无监督实践环境”，通过三个解耦阶段实现自进化，并设置“泄漏屏障”（leakage barrier）确保目标任务监督仅在最终评估阶段使用。

**技术细节（三个主要阶段）**：

1. **开放世界知识获取（Stage 1）**：
   - 使用深度研究代理（Gemini Deep Research）从开放世界检索任务相关的知识文档（背景概念、最佳实践、API文档等），形成技能知识 $k_i$。
   - 同时检索验证知识 $k_i^v$（如参考值、数据集统计不变量、交叉验证程序等），用于后续虚拟测试生成。
   - 输出：技能计划 $p_i$（包含技能架构、关键步骤、领域规则）和验证知识 $k_i^v$。

2. **无泄漏技能进化（Stage 2）**：
   - 基于计划 $p_i$ 和知识 $k_i$，基础智能体 $\pi_\theta$ 生成初始技能集 $\hat{S}_i^{(0)}$。
   - **虚拟测试生成**：利用验证知识 $k_i^v$，由一个独立的验证器LLM生成可执行的pytest套件 $\tilde{T}_i$，每个测试用例是基于可独立验证的事实（如已知行数、数值范围、输出格式）的确定性断言。
   - **迭代优化**：在沙盒环境中执行当前技能，计算虚拟通过率 $\tilde{r}^{(j)}$；若 $\tilde{r}^{(j)} < 1$，生成结构化诊断（每断言结果、根因分析、修改建议），区分“实现错误”和“知识缺口”；若是知识缺口，则触发定向检索补充知识后继续优化。最多迭代 $J=3$ 轮，或直至虚拟测试全部通过。
   - 输出：最终技能 $\hat{S}_i^*$（最后一次修改的版本，非最佳快照）。

3. **零样本目标评估（Stage 3）**：
   - 将最终技能部署到目标智能体 $\pi_{\theta'}$（可与构建智能体不同），在目标任务上执行，由隐藏的真实测试 $T_i^{GT}$ 评判通过/失败。
   - 技能是显式可迁移的制品（SKILL.md文件），无需模型特定适配。

- **公式要点**：技能构建函数 $\hat{S}_i = f(I_i, E_i, K)$，其中 $K$ 为开放世界知识；虚拟通过率 $\tilde{r}^{(j)} = \frac{1}{|\tilde{T}_i|} \sum_k \tilde{t}_{i,k}(\pi_\theta(I_i, \hat{S}_i^{(j)}, E_i))$；最终评估指标为平均通过率 $\text{PassRate} = \frac{1}{n}\sum_i T_i^{GT}(\pi_{\theta'}(I_i, \hat{S}_i^*, E_i))$。

## 6. 主要结论与发现

1. **性能优势**：OpenSkill在SkillsBench上以Opus 4.6和GPT 5.2分别达到43.6%和42.1%的平均通过率，比最强基线（Skill Creator和CoT）分别高出8.9和8.8个百分点，接近人类上界（44.5%/44.8%），并且是唯一在两个智能体上都表现稳定的方法。
2. **跨模型可迁移性**：Opus 4.6生成的技能直接应用于其他较弱模型时，OpenSkill带来5.5-14.8个百分点的提升，远优于AutoSkill（甚至低于无技能基线）和Memento，证明技能以模型无关形式编码任务知识。
3. **虚拟验证器有效性**：自建验证器与真实结果显著关联（Fisher精确检验p=0.035，点双列相关系数r=0.242，p=0.027），覆盖88.9%的人造测试意图；尽管存在一定误报和漏报，但整体信号可用于指导技能优化。
4. **开放世界知识的重要性**：消融实验显示，开放世界检索和虚拟验证器各自贡献约6个百分点，组合后额外增加2.1个百分点，二者互补。
5. **迭代次数**：过多迭代（>3）导致过拟合，性能下降。

## 8. 不足与局限

- **信息来源噪声**：论文自身指出，开放世界资源（网页、仓库）可能包含过时、矛盾或噪声信息，框架需要 provenance tracking 和来源验证，但文中未详细展开验证机制。
