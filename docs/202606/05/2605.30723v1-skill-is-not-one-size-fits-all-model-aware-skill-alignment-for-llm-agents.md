## 摘要
LLM智能体越来越多地检索外部策展的技能——在决策时检索的程序性指令——以提升长周期交互任务的性能。现有技能库通常被视为模型无关的，即在能力与行为差异显著的骨干模型间复用相同的技能表述。然而，我们在多个模型规模下的控制实验显示，技能效果强烈依赖于模型：对某一骨干模型有益的技能可能损害另一模型。受此观察启发，我们提出MASA（模型感知技能对齐）框架，该框架无需修改智能体权重即可使技能适配每个目标骨干模型。MASA分两个阶段运行：（1）分层技能进化流水线，利用爬山算法和UCB驱动的树搜索，在环境反馈和模型能力画像指导下迭代重写通用与任务特定技能；（2）在进化轨迹上训练的轻量级模型条件技能重写器，通过单次前向传播复现适配过程。在三种交互环境和四个骨干模型上的实验表明，MASA一致取得最佳整体性能，相较于最强基线提升高达25.8个点。训练后的重写器进一步泛化到未见任务和环境，无需额外搜索，以极低的推理成本持续超越更大的教师LLM。

## Abstract
LLM agents increasingly retrieve externally curated skills-procedural instructions retrieved at decision time-to improve performance on long-horizon interactive tasks. Existing skill libraries are typically treated as model-agnostic, reusing the same skill formulations across backbones with substantially different capacities and behaviors. However, our controlled experiments across multiple model scales show that skill effectiveness is strongly model-dependent: a skill that benefits one backbone can harm another. Motivated by this observation, we propose MASA Model-Aware Skill Alignment, a framework that adapts skills to each target backbone without modifying agent weights. MASA operates in two stages: (1) a hierarchical skill evolution pipeline that iteratively rewrites general and task-specific skills using hill climbing and UCB-driven tree search, guided by environment feedback and model capability profiles; and (2) a lightweight model-conditioned skill rewriter trained on evolution trajectories to reproduce the adaptation in a single forward pass. Experiments across three interactive environments and four backbones show that MASA consistently achieves the best overall performance, with gains of up to 25.8 points over the strongest baseline. The learned rewriter further generalizes to unseen tasks and environments without additional search, consistently outperforming a much larger teacher LLM at a fraction of the inference cost.

### 论文核心总结：MASA（模型感知技能对齐）

#### 1. 核心问题与整体含义（研究动机和背景）

*   **研究问题**：当前为大语言模型（LLM）智能体构建的技能库（Skill Library）通常被视为 **“模型无关”（Model-Agnostic）** 的，即所有不同规模和能力的骨干模型（Backbone）共享同一套技能库。但这样做是否合理？
*   **核心发现（颠覆性洞察）**：论文通过严格控制变量实验发现，**技能的表述形式（如详细程度）对模型效果的影响是高度模型依赖的**。对一个大模型有益的详细技能，可能会损害一个小模型的性能；甚至对于同一参数规模但来自不同模型家族的模型，其最优技能表述也完全不同。
*   **整体含义**：这个发现挑战了现有技能库设计的“一刀切”（One-Size-Fits-All）思想。它揭示了技能内容的有效性不仅仅在于“教什么”，还在于“如何教”才能与该模型的认知偏好和能力边界相匹配。一个设计不当的技能，不仅无法帮助模型，反而会成为干扰，抵消模型自身的固有优势。

#### 2. 论文提出的方法论：核心思想、关键技术细节

*   **方法论名称**：**MASA（Model-Aware Skill Alignment，模型感知技能对齐）**。
*   **核心思想**：将技能对齐视为一个**分层搜索问题**，在**环境反馈**的指导下，为每个目标骨干模型（Backbone）迭代地优化和定制技能表述。优化过程不修改模型权重，而是改变技能文本。
*   **关键技术细节（两个阶段）**：
    1.  **阶段一：分层模型条件技能进化（Hierarchical Model-Conditioned Skill Evolution）**
        *   **目标**：使用一个更强的教师LLM（Teacher LLM，如DeepSeek-V4-Pro）来迭代地优化技能，优化过程由目标模型的能力画像（Model Card）引导。
        *   **分层进化**：
            *   **通用技能（General Skills）优化**：采用**爬山法（Hill Climbing）**。每次迭代，教师模型分析目标模型的失败轨迹，然后结合模型卡（Model Card）和过往最优技能，生成一个新版本。只有在新版本带来性能提升时才被接受。
            *   **任务特定技能（Task-Specific Skills）优化**：采用**UCB驱动的树搜索（Tree Search）**。为每个任务类型独立地构建一棵搜索树，每个节点代表一个技能集，边代表教师模型的改写。通过UCB1算法平衡探索与利用，找到最优的技能表述。
    2.  **阶段二：模型条件技能重写器（Model-Conditioned Skill Rewriter）**
        *   **目标**：将昂贵的分层进化过程**摊销（Amortize）** 为一个轻量级的、单次前向传播（Single Forward Pass）的模型。
        *   **训练**：使用阶段一产生的进化轨迹（输入技能 -> 最优技能）作为训练数据，训练一个小型LLM（如Qwen3-4B）作为重写器。输入是模型卡、输入技能集和任务描述，输出是与该模型最适配的技能。
        *   **推理**：部署时，无需任何环境交互或迭代搜索，直接通过重写器一步生成适配技能。

#### 3. 实验设计：数据集/场景、Benchmark、对比方法

*   **实验场景**：
    1.  **ALFWorld**：文本家庭任务（6个子任务）。
    2.  **WebShop**：模拟在线购物。 
    3.  **Search-augmented QA**：搜索增强的问答（7个数据集，包括单跳和多跳）。
*   **骨干模型**：Qwen3 家族的变体：4B、8B、14B、32B (非思考模式)。
*   **教师模型**：DeepSeek-V4-Pro。
*   **对比方法（Baselines）**：
    1.  **No Skill**：无任何技能。
    2.  **Base Skill**：初始的、未经适配的通用技能库。
    3.  **DS-Adapter**：使用教师模型（DeepSeek-V4-Pro）进行一次性改写，但无迭代搜索。
*   **评估方式**：主要采用**成功率（Success Rate, SR）** 和交互步数（Steps）作为衡量标准，并引入了**调整奖励（Adjusted Reward）** 来平衡任务完成与无效行为。

*   **模型范围有限**：研究目前主要集中在**Qwen3家族**，虽然在附录中用Gemma3家族做了部分验证，但并未覆盖更多主流的大型模型家族（如Llama 3, Mistral, GPT-4o等）。
*   **计算成本高昂**：
    *   **进化阶段**：为每个目标模型进行层级搜索（需要数百次环境交互）的成本非常高，尤其对于闭源模型（需要调用API），可能导致实际应用中的开销难以接受。

*   **依赖自动奖励信号**：进化过程依赖于环境提供的自动成功/失败信号。对于许多没有明确定义成功标准的开放域任务，这种方法难以直接应用。

*   **未解决的根本问题**：虽然MASA有效，但它并没有解释“为什么”不同模型偏好不同的技能风格。这仍然是未来研究的一个开放性问题。
