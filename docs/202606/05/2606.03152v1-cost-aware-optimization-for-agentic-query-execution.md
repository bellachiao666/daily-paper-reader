## 摘要
经典查询优化搜索仅在成本上不同的代数等价计划。一旦引入基于大语言模型（LLM）的算子，这一假设便不再成立：其放置、顺序和粒度共同决定了美元成本和答案质量，而替代方案中的正确选择通常仅在运行时才显现。我们形式化地将此设定定义为智能体查询执行，这是一种将基于智能体的规划与执行交织在一起的查询执行范式，其中智能体工作流优化成为经典查询优化的对应物。接着，我们提出EnumGRPO，一种针对此设定的自改进优化器。在学习阶段，EnumGRPO枚举查询计划，涉及执行范式、算子类型、算子放置、选择性范围、投影宽度等决策，然后通过上下文强化学习将质量-成本反馈提炼为可重用的规划启发式。在SWAN的四个数据库中，EnumGRPO以每个查询0.011美元的LLM算子成本实现了35.4%的执行准确率，相比混合查询基线成本降低约317倍，同时答案准确率相对提升18%。

## Abstract
Classical query optimization searches over algebraically equivalent plans that differ only in cost. This assumption breaks once LLM-backed operators enter the picture: their placement, ordering, and granularity jointly determine both dollar cost and answer quality, and the right choice among the alternatives is often revealed only at runtime. We formalize this setting as agentic query execution, a query execution paradigm in which agent-based planning is interleaved with execution, and agent workflow optimization becomes the analogue of classical query optimization. We then present EnumGRPO, a self-improving optimizer for this setting. During a learning stage, EnumGRPO enumerates query plans over decisions such as execution paradigm, operator type, operator placement, selectivity scope, and projection width, then distills quality-cost feedback into reusable planning heuristics via in-context reinforcement learning. Across four databases in SWAN, EnumGRPO achieves 35.4% execution accuracy at $0.011 per query in LLM-operator cost, a ~317x cost reduction over the hybrid query baseline with an 18% relative improvement in answer accuracy.

### 1. 论文的核心问题与整体含义（研究动机和背景）

- **背景**：经典查询优化假设所有候选计划代数等价，仅在执行成本上有差异，因此优化目标是在等价计划空间中选取最低成本的计划。然而，当数据库引入基于大语言模型（LLM）的语义算子后，这一假设被打破：LLM 算子是近似的，其**放置、顺序和粒度**共同影响答案质量和美元成本，且最优方案往往在运行时通过观察中间结果才能确定。
- **核心问题**：如何对融合 SQL 和 LLM 算子的“智能体查询执行”（Agentic Query Execution）范式进行代价感知的优化？这不仅是将语义算子加入 SQL 的问题，更涉及**在运行时动态决定何时调用 LLM、调用何种算子、处理多少数据**，从而在质量与成本之间取得最佳平衡。

### 2. 论文提出的方法论：核心思想、关键技术细节、算法流程

- **核心思想**：将智能体查询执行中的计划优化形式化为**智能体工作流优化（Agent Workflow Optimization）**，并在学习阶段通过结构化枚举覆盖五个正交的决策轴，然后利用群体相对策略优化（GRPO）的对比信号，通过**上下文强化学习（In-Context Reinforcement Learning, ICRL）** 将工作流反馈提炼为可重用的自然语言规划启发式，存储在经验池中，在查询时指导智能体进行步骤式决策，无需再枚举。
- **关键技术细节**：
  1. **计划空间五轴**：
     - **执行范式**：数据驱动（Data-driven，逐行调用 LLM）vs. 代码驱动（Code-driven，LLM 合成 SQL 规则一次执行）。
     - **算子类型**：标量（Scalar，逐行推理）vs. 聚合（Aggregate，批量推理）。
     - **算子放置**：预聚合（Pre-aggregation，在 SQL 聚合前运行 LLM）vs. 后聚合（Post-aggregation）。
     - **选择性范围**：全量（Full，所有行送入 LLM）vs. 目标性（Targeted，先用 SQL 过滤再送入 LLM）。
     - **投影宽度**：窄（Narrow，仅传关键列）vs. 宽（Wide，传更多上下文列）。
  2. **EnumGRPO 算法流程**（算法1）：
     - **学习阶段**：每次处理一批查询（batch size B=2），对每个查询通过计划枚举器生成 N=5 种不同的轴值分配（确保多样性），并行执行各工作流，计算多目标奖励（组合 EX、TupleF1、CellF1 及归一化成本），在查询组内计算组相对优势（$A_i = (r^*_i - \mu)/\sigma$），然后通过四阶段经验蒸馏管道（轨迹摘要 → 对比分析 → 池更新 → 批量整合）将对比信号转化为文本启发式，更新经验池 $\varepsilon$。
     - **查询时阶段**：将学习到的经验池 $\varepsilon$ 预置到智能体系统 prompt 中，智能体基于启发式进行逐步规划与执行，无需枚举。
  3. **多目标奖励**：为缓解稀疏奖励，分别定义 Execution Accuracy (EX)、Tuple-level F1、Cell-level F1，并融合归一化成本（$\rho_i = w_{token}\times... + w_{step}\times...$），最终奖励 $r^*_i = r_i - w_{cost}\cdot\rho_i$。
  4. **上下文强化学习（ICRL）**：模型权重冻结，所有策略改进通过修改 prompt 上下文（经验池）实现，避免重训或微调 LLM，便于跨模型迁移和长期维护。

### 6. 论文的主要结论与发现

- EnumGRPO **在所有质量指标上优于对比方法**：35.4% 执行准确率，相比 Agentic Text2SQL（19.2%）相对提升 84%，相比 Agentic BlendSQL（30.0%）相对提升 18%。

- **效率提升**：平均工作流步骤从 17.05 降至 15.79，延迟从 235s 降至 153s。
- **跨模式迁移有效**：留出变体保持准确率（32.9% vs 32.1% EX）且更高效（110s vs 235s），说明启发式具有通用性。

- **经验池收敛**：池大小在学习约 16 批后稳定在 32 条，其中“执行范式”和“选择性范围”占据主导（53%），反映它们是成本杠杆的核心。

### 7. 优点

- **方法论创新**：将经典查询优化中的等价变换思想扩展到非等价的 LLM 算子计划空间，提出了五个正交决策轴，并利用**结构化枚举+组相对优势**实现高效的信用分配。
- **无需梯度更新**：采用上下文强化学习，所有改进通过 prompt 文本实现，避免了昂贵的参数重训，使优化器可跨 LLM 版本迁移，且启发式可读、可审计。
- **多粒度奖励设计**：缓解二元精确匹配带来的稀疏性，使模型能从部分正确的工作流中学习。
- **实验全面**：覆盖多数据库、多规模、留出测试，并分析了内部启发式结构，验证了方法的有效性。
- **实际可用性**：学习成本在 1460 次查询后可回本，且运行时几乎无额外开销，适合实际部署。

### 8. 不足与局限

- **实验规模局限**：仅使用 SWAN 的 120 个查询（40 学习 + 80 评估），数据库领域和查询类型有限，泛化性需更广泛验证（如 TPC-DS 风格的语义推理）。
