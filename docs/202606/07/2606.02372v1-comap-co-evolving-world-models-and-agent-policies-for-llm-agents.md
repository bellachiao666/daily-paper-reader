## 摘要
为语言智能体配备世界模型使其能够预判环境动态并在执行前评估候选动作。然而，现有文本世界模型通常在训练后固定不变，无法适应由演化中智能体产生的策略内状态-动作分布。同时，智能体改进方法往往依赖外部奖励或验证器，限制了其在真实交互环境中的适用性。本文提出COMAP，一种通过闭环交互共同演化文本世界模型与智能体策略的新框架。在每个决策步骤，世界模型预测候选动作的未来状态反馈，智能体通过评估该反馈的可靠性并相应优化动作，进行面向未来的反思。由此产生的策略内轨迹随后通过自蒸馏用于更新世界模型，使其更好地匹配智能体不断演化的交互分布。在具身任务规划、网络导航和工具使用基准测试中，COMAP持续优于竞争基线，例如在Qwen3-4B上相对提升16.75%。进一步分析表明，共同演化循环随时间提高了世界模型的预测精度，并带来更有效的长时程决策。我们的代码开源在：https://github.com/loyiv/CoMAP。

## Abstract
Equipping language agents with world models enables them to anticipate environment dynamics and evaluate candidate actions before execution. However, existing textual world models are typically fixed after training, preventing them from adapting to the on-policy state-action distributions induced by an evolving agent. Meanwhile, agent-improvement methods often rely on external rewards or verifiers, limiting their applicability in realistic interactive environments. In this paper, we propose COMAP, a novel framework that co-evolves textual world models and agent policies through closed-loop interaction. At each decision step, the world model predicts future state feedback for candidate actions, and the agent performs future-aware reflection by estimating the reliability of this feedback and refining its action accordingly. The resulting on-policy trajectories are then used to update the world model via self-distillation, allowing it to better match the agent's evolving interaction distribution. Across embodied task planning, Web navigation, and tool-use benchmarks, COMAP consistently outperforms competitive baselines, e.g., +16.75% relative improvement with Qwen3-4B. Further analyses show that the co-evolutionary loop improves the world model's prediction accuracy over time and leads to more effective long-horizon decision-making. Our code is available at: https://github.com/loyiv/CoMAP.

## 2. 方法论：核心思想、关键技术细节

- **核心思想**：通过**闭循环协同演化**（Co-Evolving）同时优化世界模型和智能体策略，两个组件通过持续交互互相促进。

- **关键技术细节**：
    1. **未来状态想象（Future State Imagination）**：智能体首先生成草稿动作 \(a_t^{draft}\)；世界模型预测其一步后果 \(\hat{s}_{t+1} = W_\phi(s_t, a_t^{draft})\)。
    2. **未来感知反思（Future-Aware Reflection）**：智能体基于当前状态、草稿动作和预测未来状态，生成**优化动作** \(a_t^{ref}\) 和**优化概率** \(p_t\)，通过门控机制决定是否替换草稿动作。
    3. **初使化训练**：使用专家示范数据对策略进行监督微调（SFT），使用真实环境转移数据对世界模型进行预热。
    4. **世界模型自蒸馏（On-Policy Self-Distillation for WM）**：世界模型分为**学生模式**和**教师模式**（EMA更新）。学生生成预测 \(\hat{s}_{t+1}^S\)，教师利用真实下一状态 \(s_{t+1}^{env}\) 提供软目标，通过**token级KL散度**计算自蒸馏损失，联合真实状态交叉熵训练。
    5. **策略演化**：利用门控未来状态信号（世界状态门控选择学生预测或真实状态）训练反思头，损失包含草稿模仿、优化动作预测和优化概率BCE。
    6. **协同演化训练闭环**：策略草稿→世界模型预测→策略反思→执行→环境转移→更新世界模型→更新策略。门控机制（世界状态门控+动作门控）抑制早期噪声，提供课程学习。

## 6. 主要结论与发现

- **主要结论**：COMAP在所有四个基准上**持续显著优于**所有竞争基线，尤其在小模型上收益更大（Qwen3-4B平均成功率从51.52%提升至60.15%，相对 +16.75%）。

## 7. 优点（亮点）

- **创新性**：首次提出**端到端闭循环协同演化**框架，将世界模型与策略视为对等伙伴而非单独优化，解决了静态世界模型的关键局限。
- **技术优雅性**：双门控机制（世界状态门控+动作门控）形成课程学习，有效抑制早期噪声；自蒸馏利用软目标提供密集监督。
- **实验扎实**：多项消融、动态分析、缩放测试充分验证各组件有效性；在多个领域和多种backbone上验证泛化性。
- **实用性强**：不依赖外部奖励或人工验证器，适用于真实交互环境；推理时只需一次一步想象，成本可控。

## 8. 不足与局限

- **文本表示假设**：假设环境状态和动作结果可用文本表示，无法直接处理视觉、空间、听觉等多模态信号。
- **推理延迟**：需要**一次额外的世界模型推理**才能进行反思（比普通ReAct多一次LLM调用），在延迟敏感应用中可能受限。
- **风险提醒**：世界模型预测可能不准确，若部署到开放世界工具或机器人系统可能产生不可靠动作，需安全审查。
- **未完全排除偏差**：初始专家示范数据可能引入人类偏差；自蒸馏可能放大早期错误。
