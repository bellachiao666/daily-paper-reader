## 摘要
系统提示优化可以在不修改底层模型的情况下改善智能体行为，生成人类可读的、与模型无关的指令。现有方法构建一个提示代理来优化任务智能体的系统提示，但提示代理自身的系统提示仍由人工设计且固定不变。我们提出自演化提示优化（SePO），将提示代理自身的系统提示与任务智能体的系统提示一同作为优化目标。SePO采用自指涉设计：单个提示代理在开放式演化搜索中同时优化任务智能体的系统提示和自身提示，该搜索维护一个候选提示存档作为垫脚石。训练分为两个阶段：预训练在多任务池上演化提示代理，微调则将其应用于目标任务。在涵盖数学（AIME'25）、抽象推理（ARC-AGI-1）、研究生级别科学（GPQA）、代码生成（MBPP）和逻辑谜题（数独）的五个基准测试中，SePO始终优于Manual-CoT、TextGrad和MetaSPO，与Manual-CoT相比平均准确率提升4.49个百分点。预训练中的提示优化技能也能泛化到预训练混合之外的任务，而非记忆每个任务的提示。

## Abstract
System prompt optimization improves agent behavior without modifying the underlying model, yielding human-readable, model-agnostic instructions. Existing methods build a prompt agent that refines task agents' system prompts, yet leave the prompt agent's own system prompt hand-engineered and fixed. We propose Self-Evolving Prompt Optimization (SePO), which treats the prompt agent's own system prompt as an optimization target alongside task agents' system prompts. SePO adopts a self-referential design. A single prompt agent improves both task agents' system prompts and its own under an open-ended evolutionary search that maintains an archive of candidate prompts as stepping stones. Training proceeds in two stages: pre-training evolves the prompt agent on a multi-task pool, and fine-tuning then applies it to a target task. Across five benchmarks spanning math (AIME'25), abstract reasoning (ARC-AGI-1), graduate-level science (GPQA), code generation (MBPP), and logic puzzles (Sudoku), SePO consistently outperforms Manual-CoT, TextGrad, and MetaSPO, improving the average accuracy by 4.49 points compared to Manual-CoT. The prompt optimization skill from pre-training also generalizes to tasks beyond the pre-training mixture, rather than memorizing per-task prompts.

## 一、论文的核心问题与整体含义（研究动机和背景）

- **核心问题**：现有系统提示优化方法（如TextGrad、PromptBreeder、MetaSPO）都构建一个“提示代理”（prompt agent）来改进任务智能体的系统提示，但该提示代理自身的系统提示是**手工设计的且固定不变**。这导致优化能力受限于人类手工工程的水平，无法随经验积累而提升，也无法跨任务泛化。
- **研究背景**：智能体性能可通过多种方式提升（重训练、设计工作流、优化系统提示等）。系统提示优化因无需修改模型权重、产生产可读且模型无关的指令而受关注。但已有方法均未将提示代理本身作为可优化对象，形成了一个不对称的优化瓶颈。
- **整体含义**：论文提出**自引用设计**——让提示代理也能像改进任务智能体一样改进自身的系统提示，从而闭合优化循环。这一设计将提示优化从一个固定工具转变为可不断积累经验的可学习技能，代表了向自演化智能体方向迈出的重要一步。

## 二、论文提出的方法论

### 核心思想
采用**自引用（self-referential）框架**：将提示代理自身视为一个特殊的任务智能体，其任务是改进任务智能体的提示。因此，同一个优化过程（开放式进化搜索）同时适用于优化任务智能体的系统提示（p）和提示代理自身的系统提示（\~p），形成闭环。

### 关键技术细节
1. **形式化定义**：
   - 任务 T = (D, S)，其中D是数据集，S是评分函数。
   - 任务智能体 A(p, M, W)，包含系统提示 p、底层语言模型 M、工作流 W。
   - 提示代理作用：输入为 (T, A, E)（任务、待优化智能体、评估结果），输出优化后的提示 p′。
   - 标准优化目标：p* = argmax acc(A; T)。
   - 自引用优化目标：\~p* = argmax E[ \~S(T, A, E, \~A(\~p)(...)) ]，与式(1)形式相同，任务不同。
2. **开放式进化搜索**（基于 Zhang et al., 2026）：
   - 维护一个候选提示的**存档（archive）**作为垫脚石。
   - **父代选择**：基于软max，带有温度自适应和子代数量惩罚，平衡开发与探索。
   - **子代生成**：提示代理根据当前提示和评估结果生成改进候选。
   - **评分**：子代相对于父代的准确率增量（或其安全变体 RER）。
   - **存档准入**：仅当子代得分不低于父代（考虑噪声裕度）时加入存档。
3. **两阶段训练流程**：
   - **预训练**：在多任务池上自演化提示代理自身的系统提示，从种子 \~p(0) 进化到 \~p*。此阶段使用自引用闭环，提示代理改进自身副本。
   - **微调**：固定已演化的提示代理（使用 \~p*），优化目标任务上任务智能体的系统提示 p，同样采用开放式进化搜索。
4. **SePO-Specialist vs SePO-Generalist**：
   - **SePO-Specialist**：预训练使用单任务（与微调目标相同）。
   - **SePO-Generalist（默认）**：预训练使用多任务混合（由贪心启发式选择，见算法2）。多样性预训练提升鲁棒性和泛化，且一次预训练可复用于所有微调任务。

### 关键算法流程（文字说明）
- **算法1**：Self-Evolving Prompt Optimization
  1. 初始化存档 A = {p(0)}。
  2. 循环 G 代，每代 K 个并行子代：
     - 从 A 中采样父代 p。
     - 通过提示代理生成子代 p′（预训练阶段提示代理用当前 \~p，微调阶段用固定 \~p*）。
     - 计算子代得分，若满足准入条件则加入 A。
  3. 返回存档中得分最高的提示 p*。

## 六、论文的主要结论与发现

1. **性能优势**：SePO-Generalist 在所有五个任务上均取得最佳准确率，平均 76.38%，比 Manual-CoT（71.89%）提升 4.49 个百分点；比 TextGrad 和 MetaSPO 分别提升 5.99 和 5.06 个百分点。
2. **自引用设计有效**：SePO-Specialist 也优于所有基线，但 SePO-Generalist 进一步提升了 2.29 个平均点，说明多任务预训练带来的泛化能力是关键。
3. **跨任务泛化成功**：即使预训练混合中不含任何与目标相关的任务（如数独从未出现），SePO-Generalist 仍将准确率从 96.95% 提升至 99.90%；在所有“无相关任务”设置下均超越 Manual-CoT，证明学到的不是记忆具体任务，而是通用的提示优化技能。
4. **消融实验证实每个组件必要**：去除自改进使平均准确率下降 1.44 点（对 ARC-AGI-1 影响最大），去除开放式进化下降 3.74 点（对 AIME'25 影响最大）。

6. **成本效益**：SePO-Generalist 分摊预训练成本后，每个任务的平均训练成本（约 $7.43）与 TextGrad（约 $0.15~$1.73）相比略高但在可接受范围，且性能显著更好。推理成本与所有方法相同（仅运行任务智能体）。

## 七、优点

## 八、不足与局限

1. **搜索深度有限**：当前仅使用 5 代、每代 2 个子代，初步扫描显示更深的搜索带来的收益并非指数级增长，可能存在性能天花板（受限于底层模型能力）。
2. **任务覆盖仍有限**：五个基准测试虽覆盖多领域，但未包含工具使用智能体、多轮对话、长程规划等更复杂场景，泛化性未完全验证。
3. **模型依赖**：仅测试了两个提示代理模型（Gemini 3.1 Pro Preview, Claude Opus 4.6）和两个任务智能体模型（DeepSeek-V3.2, Gemini 3.1 Flash-Lite Preview），未在开源模型或更大规模模型上验证。
4. **任务选择启发式**：贪心任务选择方法简单，论文承认可被更先进的多任务选择算法替代，当前设定的泛化性可能受限。
5. **潜在灾难性遗忘**：预训练学习到的通用技能可能在微调过程中被覆盖或弱化（虽然实验显示保留良好，但未系统分析）。
6. **评估噪声**：使用少量训练样本（每批次16个，1:1失败/成功比）评估候选提示，可能对评分稳定性有影响（论文使用噪声裕度 ε=0 未详细讨论）。

8. **实际部署成本**：虽然 API 计费较低，但若自建模型运行进化搜索，GPU 时间可能显著增加，论文未提供本地资源数据。
