## 摘要
自主LLM训练常被构架为配方搜索，这使训练框架基本保持静态。这一局限在智能体强化学习中尤为突出，其中变化的瓶颈和标量奖励掩盖了多种失败模式。我们提出EvoTrainer，一个通过经验反馈协同进化LLM策略和训练端框架的自主训练框架：它诊断轨迹级证据、修正诊断、回测干预并积累可复用技能。在数学推理、竞赛编程代码生成和仓库级软件工程上的评估表明，在相同数据、代码库和评估协议下，EvoTrainer达到或超过人工设计的RL参考基线，其中在长周期智能体SWE任务上增益最大。轨迹分析显示，保留的策略会因领域而异，进化的诊断阻止无效的高分分支被提升，可复用技能塑造后续搜索。自主LLM RL应超越配方搜索，走向策略与解释它们的训练框架的联合进化。

## Abstract
Autonomous LLM training is often framed as recipe search, which leaves the training harness largely static. This limitation sharpens in agentic RL, where shifting bottlenecks and scalar rewards mask diverse failure modes. We introduce EvoTrainer, an autonomous training framework that co-evolves LLM policies and training-side harnesses through empirical feedback: it diagnoses rollout-level evidence, revises diagnostics, backtests interventions, and accumulates reusable skills. Evaluated on mathematical reasoning, competitive-programming code generation, and repository-level software engineering, EvoTrainer matches or exceeds the human-engineered RL references under the same data, codebase, and evaluation protocol, with the largest gain on long-horizon agentic SWE. Trajectory analyses show that retained strategies diverge across domains, evolving diagnostics prevent invalid high-scoring branches from being promoted, and reusable skills shape later search. Autonomous LLM RL should move beyond recipe search toward joint evolution of policies and the training harnesses that interpret them.

## 1. 核心问题与整体含义（研究动机和背景）

- **研究动机**：当前自治LLM训练通常被建模为“配方搜索”（recipe search），即仅在模型权重、数据、超参数等配方层面进行迭代，而训练框架（诊断指标、分析工具、干预逻辑等）基本保持静态。这种静态框架在智能体强化学习（Agentic RL）中尤为不足，因为：
  - 主导瓶颈会动态变化（从奖励稀疏 → 行为崩塌 → 评估伪影 → 低信息轨迹组等）。
  - 标量验证分数可能掩盖多种失败模式（如奖励泄露、不健康行为、低效搜索等）。
  - 后期版本可能需要早期阶段不存在的诊断工具。
- **整体含义**：论文提出应将训练系统本身作为改进对象，实现“策略”与“训练框架”的协同进化（co-evolution），从而突破单一配方搜索的局限。

## 2. 方法论：核心思想、关键技术细节

### 核心思想
- 将自主训练视为一个**证据条件化的版本转换过程**，其中策略版本 \(v_i\) 和训练框架 \(h_i\) 共同进化。
- 训练器智能体（trainer agent）在每次训练完成后，分析轨迹证据，诊断问题，更新诊断框架，提出干预，并决定下一个版本的方向。

### 关键组件与流程（参见图2）

#### 内循环：策略自我进化（Policy Self-Evolution）
- 每个版本 \(v_i\) 生成一组候选分支 \(B_i = \{b_i^{(1)}, ..., b_i^{(K)}\}\)，每个分支对当前基线实施一个干预 \(\Delta_i^{(k)}\)（如修改奖励、过滤、数据、配置等）。
- 默认采用**单因素干预**以保证归因清晰；若需捆绑更改，需记录理由并在后续通过消融/回测验证。
- 分支通过 Git worktree 隔离实现并行训练。
- 训练完成后，根据证据决定保留、剪枝、回退或合并分支，形成新的基线 \(v_{i+1}\)。

#### 外循环：训练器反思与框架进化（Trainer Reflection & Harness Evolution）
- 诊断框架组织为四个互补层：
  - **分数层**（Score）：验证指标、基准级改进。
  - **信号层**（Signal）：奖励方差、死组比率、组件级奖励采用情况。
  - **行为层**（Behavior）：工具使用模式、搜索/编辑/测试结构、轨迹长度、退化模式、环境异常。
  - **版本层**（Version）：跨分支的晋升/拒绝/保留决策。
- 当现有证据无法支持解释结果、区分假设或选择下一个干预时，触发**诊断缺口**（diagnostic gap），并通过以下四种轴升级框架：
  - 指标扩展（如死组比率、轨迹多样性、首次编辑轮次）。
  - 分析器专门化（领域特定分析例程）。
  - 流程修订（如从分数优先转向行为优先分析）。
  - 外部证据检索（查询论文、代码库、先前轨迹以合成候选干预或框架更新）。

#### 持久记忆与可重用技能库
- 存储：版本账本（配置、Git diff、决策）、案例记忆（重复模式）、技能库（已验证的分析器、修复策略、流程模板）、搜索追踪（检索查询与外部来源）。
- 技能可跨领域重用（例如 SWE 中开发的 StdGroupFilter 被后续用于 Math 和 Coding）。

#### 自主性范围
- 人类：引导工作空间、启动首个可运行版本、审批高成本/关键性执行（如训练启动、版本晋升）。
- 训练器智能体：自主完成诊断循环、框架升级、技能创建、外部搜索、回测、干预提案。

#### SWE 训练核心实例化（GRPO 风格）
- 基于 GRPO，使用非对称 Clip-Higher 边界和弱 KL 正则化。
- 组相对优势 \(A_i = (r_i - \mu_g) / (\sigma_g + \epsilon)\)。
- 奖励设计（进化后的最终版本）：
  \[
  r_i = 1.0 \cdot CR_i + 0.1 \cdot IF_i + 0.1 \cdot SBE_i + 0.15 \cdot ETT_i
  \]
  （CR：隐藏测试正确性；IF：指令遵循奖励；SBE：搜索后再编辑行为；ETT：编辑后再测试行为）
- 方差感知分组过滤：移除零方差组（死组），自适应过滤低信息组。

## 6. 主要结论与发现

1. **整体表现**：EvoTrainer 在所有领域均超越无 RL 基线，并达到或超过人工设计 RL 最佳配置。
   - SWE-9B：38.16 BC% vs 人工 33.77（∆=+4.39, p<0.001）
   - Math 聚合：79.49 vs 人工 76.60（∆=+2.88, p<0.001）
   - SWE-4B 和 Coding 在置信区间内与人工匹配（p>0.1）。
2. **跨领域策略分化**：保留的配方在不同领域差异显著（Math 引入代码解释器，Coding 引入双重过滤，SWE 使用行为敏感奖励），表明 EvoTrainer 适应领域特定瓶颈。
3. **训练框架进化比分数驱动迭代更强**：
   - SWE-9B 分数主导子路径饱和于 33.33 BC%，而引入丰富诊断后推进至 38.16（∆=+4.83）。
   - 框架审计阻止了因 Git 泄漏导致的高分分支被错误晋升。
4. **技能库提供跨领域重用**：SWE 开发的 StdGroupFilter 被 Math 和 Coding 重用，显著改善死组比率和最终性能。
5. **自主 LLM RL 应向策略与训练框架的联合进化发展**。

## 7. 优点

### 方法亮点
- **双循环进化框架**创新性地将训练框架本身作为进化对象，突破传统配方搜索。
- **诊断四层架构**（分数/信号/行为/版本）系统化，覆盖常见失败模式。
- **版本控制纪律**（单因素干预、Git worktree、保留/剪枝/合并决策）使实验可审计、可回溯。
- **持久记忆与可重用技能库**促进跨领域知识迁移，避免重复实验。
- **人类与训练智能体分工明确**（人类引导环境、审批高成本动作；智能体自主诊断和提案），兼顾自动化与可控性。

### 实验亮点
- 在三个代表性领域（单回合→长周期智能体）系统评估，验证方法泛化性。
- 对比基线全面（无 RL、自主仅分数、多种算法变体、人工最佳配置）。
- 统计检验严谨（paired bootstrap + Wilcoxon，提供置信区间和 p 值）。
- 自然反事实分析（无需单独消融实验）有效隔离各组件贡献。

## 8. 不足与局限

### 实验覆盖与统计
- **单训练种子**：所有版本只在 seed=42 下训练一次，无法量化训练随机性。作者指出受限于算力（common in large-scale LLM-RL practice）。
- **版本轨迹长度有限**：每领域仅 7–10 个版本，未探索足够长的进化行为（如技能库和案例记忆需要主动修剪或层次组织的场景）。
- **无多域交叉验证**：三个域任务性质差异大，但未在所有任务上重复相同消融模式（如 SWE-4B 和 Coding 无正式消融表格）。

### 方法局限
- **依赖强大训练器模型**：实验中使用 Claude Sonnet 4.6，若训练器能力不足，诊断和干预质量可能下降。
- **人类审批仍为必要环节**：高成本训练和版本晋升由人类把关，限制了完全自主性。
- **诊断框架进化成本高**：训练器标记消耗 \(4\times 10^8\) tokens，对 token 预算敏感。
- **单因素干预原则可能导致空间受限**：某些情况下捆绑干预可能更高效，但记录理由和后续消融增加复杂性。

### 公平性
- 人工基线虽由同一团队开发，但版本数量更多（SWE 约 23 vs EvoTrainer 15），仅从 GPU 小时看 EvoTrainer 更高效，但人工基线可能包含更多失败经验的隐性知识。
- 未与最新非公开最佳 RL 方法（如某些闭源系统）比较，基准线为“团队内人工最优”，可能未达领域上限。
