## 摘要
基于LLM的智能体系统的自动化设计与优化，能够构建出比现成智能体模式显著提升结果质量的复杂系统。然而，对已部署的智能体系统的研究表明，生产系统更关注推理成本的简单性、可控性和可预测性等问题。本文提出了设计和优化实用智能体系统的原则性方法。我们描述了一个智能体框架，该框架通过定义在受限上下文中递归调用LLM的“伪工具”，使设计者能够强制智能体系统的模块化。利用该框架，我们为多种任务手工设计智能体，并证明与动态规划的工作流相比，手工构建的固定工作流通常更便宜且更准确。随后，我们提出了针对该框架所需智能体组件（即伪工具和固定工作流）的新型学习方法。这些学习方法通常优于手工设计的智能体。我们还利用该框架的模块化性，应用多目标优化方法联合优化成本与响应质量，并融合多个学习系统的结果。

## Abstract
Automated design and optimization of agentic LLM-based systems leads to sophisticated systems that substantially improve result quality over off-the-shelf agentic patterns. However, studies of fielded agentic systems show that production systems focus much more on issues such as simplicity, controllability, and predictability of inference costs. In this paper we propose principled approaches to designing and optimizing practical agentic systems. We describe an agent framework that enables designers to enforce modularity in agentic systems, by defining "pseudo-tools" that call LLMs recursively on a restricted context. Using this framework we hand-engineer agents for a diverse set of tasks, and show that relative to dynamically-planned workflows, hand-constructed fixed workflows are generally cheaper and more accurate. We then propose novel learning methods for the agentic components required by this framework, namely pseudo-tools and fixed workflows. These learning methods generally outperform hand-engineered agents. We also exploit the modularity of the framework to apply multi-objective optimization methods to jointly optimize cost and response quality and blend the results of multiple learning systems.

## 2. 论文提出的方法论
### 核心思想
- **模块化通过“伪工具”（Pseudo-tools, ptools）实现**：伪工具是类似于外部LLM工具的组件，但在被调用时以受限上下文递归调用另一个LLM，从而将复杂的推理过程分解为可复用、可独立优化的子模块。
- **接口-实现分离**：每个接口（带有类型签名和自然语言文档的函数桩）可以绑定到不同实现（如LLM调用、Python代码、ReAct子智能体等），方便灵活配置。
- **固定工作流优于动态规划**：手工设计的固定工作流（静态Python函数）比动态ReAct循环更便宜且通常更准确。

### 关键技术细节
1. **伪工具归纳学习（Pseudo-tool Induction）**：
   - 输入：基础智能体在训练实例上的轨迹（ReAct或CoT模式）
   - 过程：四阶段LLM流水线：①标注每步动作为简短类型；②合并同义类别；③保留高频类别（最多K=5）；④为每个保留类别合成接口桩（名称、类型签名、文档字符串）
   - 输出：Python模块，每个接口绑定到单次LLM调用（使用合成的文档字符串作为提示）
   
2. **代码蒸馏（Code Distillation）**：
   - 将伪工具的实现从LLM调用替换为Python函数
   - 基于收集的输入/输出对，用强代码生成LLM生成n个候选函数，在训练集上执行并选择最佳，通过反馈迭代R轮
   - 允许函数弃权（返回None），此时回退到LLM实现；仅当保留集上错误率低时才保留蒸馏代码

3. **工作流学习**：
   - **代码蒸馏**：对根接口应用代码蒸馏，生成Python工作流编排现有工具包
   - **爬山调优（Orchestration Learner）**：联合迭代改进工作流和伪工具，由监督LLM（Gemini 3.1 Pro）提出编辑，根据训练/评估精度接受或拒绝，直到迭代预算或连续无改进

4. **多目标优化**：
   - 使用DEAP框架的NSGA-II算法，将每个接口的绑定方法（method gene）和模型（model gene）编码为染色体，在验证集上联合优化正确率和成本
   - 支持异构LLM分配（不同子接口绑定不同模型）
   - 通过缓存LLM调用降低优化成本（最多减少8倍）

1. **手工固定工作流优于动态ReAct**：在19个任务中，静态工作流平均正确率0.64（ReAct 0.56），成本0.70美元/100例（ReAct 1.66美元），即更准更便宜2倍以上。

3. **学习组件超越手工**：

4. **代码蒸馏可有效将伪工具替换为Python工具**：在7/13基准上有伪工具成功蒸馏为代码，降低成本和潜在提升正确率

6. **“奖励黑客”现象**：工作流学习者有时会生成完全跳过工具的纯Python代码（利用正则等捷径），虽在多数基准上泛化良好，但可能偏离设计意图。
