## 摘要
最近的AI智能体能够灵活调用技能来解决复杂任务，但其长期改进从根本上受限于缺乏系统性的技能构建、积累与迁移。特别是，由于缺少统一的技能整合框架，智能体倾向于在不同任务中冗余构建相似能力，无法有效将经验转化为可重用资产，并且难以将任务特定技能泛化到新场景。为解决这一局限，我们提出SkillPyramid，一种通过重用现有技能经验以实现更广泛任务泛化的技能整合框架。基于分层技能拓扑结构，SkillPyramid进一步引入自进化机制，使智能体能够在任务执行过程中组合、验证并整合新技能。在ALFWorld、WebShop和ScienceWorld上的实验表明，基于四种骨干模型，SkillPyramid将平均奖励提升了38.0%，并将执行步骤减少了27.7%。总体而言，我们的方法将技能集合从静态资源池转变为动态进化系统。

## Abstract
Recent AI agents can flexibly invoke skills to solve complex tasks, but their long-term improvement is fundamentally constrained by a lack of systematic skill construction, accumulation, and transfer. In particular, without a unified framework for skill consolidation, agents tend to redundantly construct similar capabilities across different tasks, are unable to effectively transform experience into reusable assets, and struggle to generalize task-specific skills to novel scenarios. To address this limitation, we propose SkillPyramid, a skill consolidation framework that reuses existing skill experience for broader task generalization. Operating on a hierarchical skill topology, SkillPyramid further introduces a self-evolution mechanism that enables agents to compose, validate, and incorporate new skills during task execution. Experiments on ALFWorld, WebShop, and ScienceWorld across four backbone models show that SkillPyramid substantially increases the average reward by 38.0% and reduces execution steps by 27.7%. Overall, our method transforms a skill collection from a static resource pool into a dynamic evolution system.

## 2. 论文提出的方法论

### 核心思想
- **SkillPyramid**：将技能组织成多层金字塔结构。底层（L0）保存细粒度、可重复使用的原子能力；上层（Lk）抽象出常见的问题解决模式。通过显式的重用关系和依赖建模，实现技能之间的可迁移性。

### 关键技术细节

1. **技能表示**：每个技能 s 定义为结构化的自然语言程序 s = (n, d, c)，其中 n 为技能名称，d 为简短描述，c 为技能内容（含适用条件、执行步骤、输入/输出等）。

2. **金字塔构建（Pyramid Construction）**：
   - **关系分析器（Relation Analyzer）**：对现有技能库先进行粗粒度分组（基于名称和描述），再对每组进行细粒度分析，识别出两种关系类型：向下原子提取（Downward Atomic Extraction）或向上抽象归纳（Upward Abstract Induction）。
   - **关系构建器（Relation Builder）**：
     - **向下原子提取**：从一组相关技能中提取共同的最小可重用能力，形成原子技能（atomic skill），并放到原技能下一层；原技能被重写为引用该原子技能。
     - **向上抽象归纳**：从一组相关技能中归纳出共同的解决问题模式，形成抽象技能（abstract skill），并放到原技能上一层；抽象技能记录对下层技能的引用。
   - 迭代执行最多 T 轮，直到没有新技能产生。

3. **技能创建（Skill Creation）**：
   - 给定新任务指令 x，技能创建者（Skill Creator）从当前金字塔中检索相关技能，分两步生成新技能：
     - **框架构建**：优先使用高层抽象技能勾勒任务结构和高层逻辑。
     - **细节实例化**：优先使用底层原子技能填充可执行操作、输入检查等。
   - 若现有技能完全覆盖任务，则直接复用；否则组合生成新技能 ŝ。

4. **自我进化（Self-Evolution）**：
   - 每生成一个新技能 ŝ 后，增量更新金字塔结构：关系分析器仅以 ŝ 为查询，检索可能相关的现有技能，并交由关系构建器构建新的重用关系，将更新 ∆P_t 插入到当前金字塔 P_t 中，形成 P_{t+1}。

## 6. 论文的主要结论与发现
- **SkillPyramid 显著提升性能**：平均奖励提升 38.0%（从 ReAct 的 53.4 提升至 73.7），平均步数减少 27.7%（从 20.2 降至 14.6）。
- **泛化能力突出**：在 unseen 任务上，SkillPyramid 平均奖励达 84.8，远超 ReAct+Skills（73.9）等基线，表明原子技能和抽象技能捕获了可迁移的知识。
- **交互效率提升**：步数减少明显，尤其在 ALFWorld 上，如 DeepSeek-V3.2 下 seen 任务步数从 15.6 降至 10.7。
- **消融验证**：每个组件（原子技能、抽象技能、自我进化、金字塔重用）都是必要的；从头生成技能效果最差（甚至低于扁平技能库），强调在已有层级上构建的重要性。
- **自我进化**：完全版 SkillPyramid 在后期保持稳定且高的成功率，静态版本后期退化，仅追加版本不稳定。
- **原子与抽象技能的不同作用**：原子技能对原子密集型任务（低层操作丰富）更有帮助，抽象技能对抽象密集型任务（长依赖链）更关键。
