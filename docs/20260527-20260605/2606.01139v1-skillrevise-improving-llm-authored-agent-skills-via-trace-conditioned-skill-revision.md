## 摘要
智能体技能是使LLM智能体能够执行工作流、验证约束并从失败中恢复的程序性产物。现有的自进化方法利用累积的轨迹来完善技能。然而，它们在冷启动场景中表现不佳，因为此时只有初始的不完美技能可用。因此，技能构建通常依赖于专家编写或一次性LLM生成。专家编写的技能成本高昂，且可能与LLM智能体实际执行任务的方式不一致，而一次性生成的技能可能在语法上良好，但在行为上较弱。为弥补这一差距，我们提出SkillRevise，一种基于执行轨迹的框架，旨在迭代改进这些初始技能。SkillRevise从执行证据中诊断技能缺陷，从通用记忆中检索相关修复原则，并应用基于执行的编辑。通过重新执行候选技能并测量经验效用，系统地保留最优技能版本。在三个基准测试和五个LLM上的评估显示，SkillRevise显著优于一次性基线，将基本智能体在SkillsBench上的成功率从36.05%提升至61.63%。此外，修订后的技能表现出强大的跨模型迁移能力，能够捕获超越模型特定产物的通用程序性知识。

## Abstract
Agent skills are procedural artifacts that enable LLM agents to execute workflows, verify constraints, and recover from failures. Existing self-evolving methods refine skills using accumulated trajectories. However, they struggle in cold-start settings, where only an initial, imperfect skill is available. Consequently, skill construction defaults to expert authoring or one-shot LLM generation. Expert-authored skills are costly and may not align with how LLM agents actually execute tasks, while one-shot generated skills can be syntactically well formed yet behaviorally weak. To bridge this gap, we propose SkillRevise, an execution-grounded framework designed to iteratively refine these initial skills. SkillRevise diagnoses skill defects from execution evidence, retrieves relevant repair principles from a general memory, and applies execution-anchored edits. By re-executing candidates and measuring empirical utility, it systematically retains the optimal skill version. Evaluated across three benchmarks and five LLMs, SkillRevise substantially outperforms one-shot baselines, improving the base agent's success rate on SkillsBench from 36.05% to 61.63%. Furthermore, the revised skills exhibit strong cross-model transferability, capturing generalized procedural knowledge over model-specific artifacts.

## 2. 论文提出的方法论

### 核心思想
- **以执行反馈为驱动**：将技能改进视为有界的、基于执行的修订循环，而非开放式的自进化或检索。
- **任务特定/通用分解**：从当前执行证据中**诊断**缺陷（任务特定），从**原则记忆**中检索可复用的修复模式（通用），通过**执行锚定**的编辑生成候选技能，再通过重新执行和**效用门控**选择最优版本。

### 关键技术细节

1. **诊断（Diagnosis）**  
   - 三元组：`(Vi, Ai, Ki)`，其中`Vi`为验证规范，`Ai`为失败归因，`Ki`为保留约束（哪些已通过的检查必须保持）。
   - 将原始执行证据转化为修复约束：明确要修什么、凭据什么、不能破坏什么。

2. **原则记忆（Principle Memory）**  
   - 存储**可复用的修复模式**，而非任务解决方案。每个原则包含触发条件、缺陷标签、修复规则、行动模板、验证模板、迁移约束等。
   - 初始包含**7个修复原则**（如“向验证器暴露输出”、“保留正确行为”、“检查后再行动”等）。

3. **修订算子（Revision Operator）**  
   - 输入当前技能`Si`、诊断`Di`、绑定的原则集`Pi`，输出候选修订技能`ˆSi+1`和修订痕迹`zi`（记录证据到编辑的映射、执行锚点等）。
   - **执行锚点**指定文本编辑应如何改变智能体行为（具体动作、预期证据、指令位置）。

4. **有界修订回合（Bounded Revision Episode）**  
   - 最多`B`轮（默认3轮）。每次：执行当前技能 → 诊断 → 若存在技能级缺陷则检索原则并绑定 → 修订 → 重新执行 → 效用门控更新最佳技能。
   - **效用门控**：选择所有评估过的候选技能中效用最高的，而非最后生成的版本。效用函数组合了成功率、效率、迁移和干扰惩罚（见公式9）。

1. 初始技能`S0`；设`Sbest = S0`，候选集`H = {S0}`。
2. 对每轮`i`（1到B）：

## 8. 不足与局限

- **依赖验证器反馈**：诊断和效用门控完全依赖验证器，稀疏或不对齐的验证器可能导致过拟合或修复错误。无法处理隐藏需求。
- **额外执行预算**：每次修订需要重新执行，增加了Token消耗、工具调用和延迟。虽然没有大型训练成本，但多次API调用仍可能昂贵。
- **无技能回退缺失**：默认maxrev3不包含无技能执行作为候选，在有些任务中无技能本身更好（如video-filler-word-remover案例），SkillRevise无法退回。
- **评估范围有限**：仅涵盖基于验证器的基准，不涉及长期部署、安全关键、对抗性或需要自定成功标准的环境。
- **原则记忆手动初始化**：当前7个原则由作者预设，可能遗漏某些领域特定或新兴缺陷模式。吸收机制可选且保守，但未深入评估其扩展性。
- **跨模型迁移不对称**：不同模型从同一GPT-5.5技能中获益程度不同（Kimi-2.6只有33.33%成功率），部分执行器可能因提示风格差异难以完全利用。
- **缺乏更深入的鲁棒性分析**：未测试修订技能随时间或在不同工具版本下的退化情况。
