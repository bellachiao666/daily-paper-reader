## 摘要
近期自进化智能体已表明技能可以通过执行被发现、精炼和积累。然而，现有技能进化框架通常假设固定的工具层并独立评估每项技能，限制了其修复工具级故障或推理技能间交互的能力。我们提出SkillSmith，一种协同感知的技能-工具协同进化框架。SkillSmith引入了统一提议空间，其中反思产生联合修改技能与工具的原子束，使得当技能进化识别到可重用能力缺口时，工具可以被封装、编辑、组合、拆分或淘汰。为了引导这种联合搜索，SkillSmith维护了一个受Lotka-Volterra动力学启发的生态效用模型，其中从执行轨迹估计的交互矩阵捕获技能间的互补性与冲突，并为检索、变异优先级排序和淘汰提供压力信号。此外，SkillSmith记录反模式，包括失败特征、因果归因和补救措施，以加速诊断并否决重复已知错误的提议。在包括WildClawBench的三个基准测试和五个Qwen3.5模型规模上的实验表明，SkillSmith持续优于强基线，其增益随任务复杂性和多技能共激活增加而放大。

## Abstract
Recent self-evolving agents have shown that skills can be discovered, refined, and accumulated through execution. However, existing skill-evolution frameworks typically assume a fixed tool layer and evaluate each skill independently, limiting their ability to repair tool-level failures or reason about interactions among skills. We propose SkillSmith, a synergy-aware skill-tool co-evolution framework. SkillSmith introduces a unified proposal space in which reflection produces atomic bundles that jointly modify skills and tools, allowing tools to be wrapped, edited, composed, split, or retired when skill evolution identifies a reusable capability gap. To guide this joint search, SkillSmith maintains an ecological utility model inspired by Lotka-Volterra dynamics, where an interaction matrix estimated from execution traces captures pairwise complementarity and conflict among skills and provides pressure signals for retrieval, mutation prioritization, and retirement. Furthermore, SkillSmith records anti-patterns, including failure signatures, causal attributions, and remedies, to accelerate diagnosis and veto proposals that repeat known mistakes. Experiments on three benchmarks, including WildClawBench, and five Qwen3.5 model scales show that SkillSmith consistently outperforms strong baselines, with gains that amplify as task complexity and multi-skill co-activation increase.

  - **工具层固定**：失败可能源于工具缺陷（如API过时、返回格式错误），但现有系统只能通过修改技能来“绕路”补偿，导致技能描述臃肿且易出错。
  - **技能独立评估**：每次变异只修改单个技能，假设技能价值相互独立。但实际任务中多个技能同时被检索和组合执行，存在互补或冲突的交互，独立的评估无法捕捉协同效应。
  - **失败经验不保留**：失败轨迹常在一次反思后被丢弃，只保留成功产物，导致重复诊断和无意中重建已被淘汰的技能，浪费计算资源。
- **整体含义**：为了解决以上问题，本文提出技能与工具协同进化的新范式，将智能体自我改进视为一个整合技能、工具和失败记忆的生态系统，而非孤立的策略修补。

## 2. 方法论：核心思想、关键技术细节与算法流程

### 2.1 核心思想
- **SkillSmith框架**：将智能体的外部状态表示为三元组 Σ_t = (S_t, T_t, F_t)，其中S_t为技能库，T_t为工具库，F_t为反模式记忆。通过三个关键扩展实现协同进化：
  1. 基于原子束（atomic bundle）的技能-工具联合进化
  2. 协同感知的技能生态系统动力学（Lotka-Volterra模型）
  3. 反模式记忆（anti-pattern memory）用于加速诊断和否决重复错误

### 2.2 关键技术细节

**（1）统一反思与提议规划（Unified Reflection and Proposal Planning）**
- 每次迭代从Pareto前沿采样候选配置，在训练集上执行并收集失败样本（得分低于阈值θ）。
- 反射器R输出一个原子束L，可同时包含技能和工具的修改提案。原子性保证相互依赖的变更同时生效，避免无效中间状态。
- 反射前，R接收三类辅助信号：
  - 反模式记忆检索的匹配失败记录（提供归因和补救）
  - 生态动力学提供的效用趋势Δu_i和交互估计β̂_ij（指出衰退技能或冲突配对）
  - 结构化的反馈函数μ_f（如编译错误、缺失文档报告）
- 工具修改被限制为五种类型原始操作：**WRAP**（封装，添加预处理/后处理层）、**EDIT**（编辑实现或接口）、**COMPOSE**（组合两个工具为一个）、**SPLIT**（拆分多功能工具为两个）、**RETIRE**（退役工具并记录“墓志铭”至反模式记忆）。
- 实现协同感知合并算子（crossover），将两个不同谱系的Pareto状态进行合并，合并后会检查新技能对间的生态兼容性（当β̂_ij < -β_thresh时标记为冲突，进行扩展验证）。

**（2）技能生态系统动力学（Skill Ecosystem Dynamics）**
- 将技能库视为共享生态系统，采用Lotka-Volterra竞争-共生方程建模。
  - 个体效用观测：通过执行日志计算技能激活时的标准化残差，再经EMA平滑得到û_i^(t)。
  - 协同效用估计：对于共现次数超过阈值的技能对，计算共激活残差相对于各自独立激活残差最大值的增量，正值表示互补，负值表示冲突。
  - 动态更新公式：u_i^(t+1) = u_i^(t) + ε û_i^(t) u_i^(t) [1 - (Σ_j w_ij u_j^(t))/K]，其中K为承载容量，w_ii=1，w_ij = -β̂_ij。
  - 效用用于指导检索评分、变异优先级排序和淘汰。
- 维护**实例级Pareto前沿**：每个训练实例记录哪个系统状态得分最高，非支配状态定义为至少在一个实例上表现最佳且不被其他状态严格支配。采样候选时按唯一最优实例数加权，保持战略多样性。

**（3）反模式记忆（Anti-Pattern Memory）**
- 每个记录ϕ = (p, a, c)，其中p为结构化的失败特征（包括涉及技能/工具、错误类别等），a为因果归因，c为补救措施。
- 两个机制：
  - **诊断加速**：反思前检索与当前失败特征最相似的top-k反模式，将历史归因和补救注入反射上下文，减少LLM推理步骤；实践中命中率从0%升至约35%，后阶段反射调用减少约31%。
  - **提议否决**：候选束生成后，与反模式记忆进行二次匹配，若相似度超过阈值θ_veto=0.85且记录非开放，则否决该提议并要求修订。若修订两次仍触发否决，则丢弃整个提案。实践中否决机制在三个基准上触发14次，成功阻止11个重复配置。

**（4）验证与状态更新**
- 候选Σ'经过渐进验证：工具单元测试（当包含工具操作时）→ 端到端集成测试 → 回归检查。对于合并候选，回归检查额外在双亲谱系擅长的任务类别上强制执行。
- 通过后加入Pareto前沿，更新生态观测和动态效用，记录新反模式。效用持续低于淘汰阈值连续T_ret轮的技能被移除，其信息以“墓志铭”形式存入反模式记忆。

## 6. 主要结论与发现

- **主结果**：SkillSmith在所有基准和所有模型规模上**持续优于**基线方法，且增益随模型规模增大而单调增加。在397B模型上，OfficeQA达到80.1%（相比Base Agent +18.3%），SealQA达到49.5%（+18.9%）。
- **缩放效应**：任务越复杂、步骤越多、多技能共激活率越高，SkillSmith的优势越明显。解构分析显示工具复杂度、交互密度、技能协同等因素均与增益正相关。
- **进化稳定性**：在100轮进化中，SkillSmith能够在扰动后快速恢复并持续提升，而EvoSkill（固定工具层）被卡在约30%准确率；生态控制防止库膨胀（<28组件），反模式记忆降低回归率（从7.6%降至2.1%）。
- **消融验证**：锁定工具层（技能-only）造成最大性能下降（WildClawBench上-6.8%）；移除生态模型导致库膨胀（21+7个组件）；移除反模式记忆使回归率增至7.6%。
- **协同进化有效性**：工具层修复能一次解决多个技能共享的工具缺陷（如案例7中一个WRAP+EDIT束修复三个依赖技能），避免技能层“绕路”修补。
