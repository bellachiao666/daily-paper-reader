## 摘要
上下文适应通过迭代地根据任务反馈修正可调提示，无需修改模型权重，从而自动化基于LLM的系统的提示工程。将这一范式扩展到多LLM智能体系统至关重要：现有方法存在不准确的信用分配和缺乏收敛保证的问题。我们提出了基于图的目标反向传播（GTBP），这是一个针对建模为有向无环图的智能体工作流的上下文适应框架。GTBP通过工作流图反向传播局部目标输出，并利用目标-输出差异指导阶段式提示更新机制。理论上，我们证明了GTBP的阶段式提示更新在迭代中趋于稳定，并且足够强大的LLM优化器可以降低整体目标。实验上，GTBP在三个基准测试中始终优于强基线，同时保持可比较的计算成本。

## Abstract
Context adaptation automates prompt engineering in LLM-based systems by iteratively revising tunable prompts from task feedback, without modifying model weights. Extending this paradigm to multi-LLM agentic systems is crucial: existing methods suffer from inaccurate credit assignment and lack convergence guarantees. We propose \textbf{G}raph-based \textbf{T}arget \textbf{B}ack-\textbf{P}ropagation (GTBP), a context adaptation framework for agentic workflows modeled as directed acyclic graphs. GTBP propagates local target outputs backward through the workflow graph and uses target--output discrepancies to guide a stage-wise prompt update mechanism. Theoretically, we show that GTBP's stage-wise prompt updates become stable over iterations, and that a sufficiently capable LLM optimizer can decrease the overall objective. Empirically, GTBP consistently outperforms strong baselines across three benchmarks while maintaining comparable computational cost.

## 1. 核心问题与整体含义（研究动机和背景）

- **问题背景**：在多LLM（大语言模型）智能体系统中，每个模块（agent）使用可调的提示（prompt）来指导其行为。上下文适应（context adaptation）旨在通过任务反馈自动更新这些提示，而保持模型权重冻结。这避免了昂贵的手动提示工程。
- **核心挑战**：多模块系统中存在**信用分配问题**（credit assignment）：最终的输出误差需要归因于各个模块的上下文（提示）。由于LLM模块是黑盒、文本输入输出且不可微，传统的基于梯度的信用分配（如反向传播）难以直接应用。
- **现有方法的不足**：一些方法（如GEPA、ACE）通过LLM对整个工作流进行反思来隐含地分配信用，但存在**归因模糊性**（attribution ambiguity），且缺乏理论收敛保证。TextGrad、Agentic Neural Networks等利用图结构传播文本反馈，但仍是启发式的，未与优化目标直接耦合。
- **论文目标**：提出一种有理论保证的、结构化的信用分配方法，用于多LLM智能体系统的上下文适应。

## 2. 方法论：核心思想、关键技术细节、算法流程

### 核心思想：基于图的目标反向传播（GTBP）

- 将智能体工作流建模为**有向无环图（DAG）**，每个节点代表一个LLM子模块，输出传递给下游节点，最终输出由输出节点聚合。
- 灵感来源于**差分目标传播（Difference Target Propagation, DTP）**：通过传播局部目标（local targets）而非梯度来分配信用。
- **关键流程**（见图1和算法1-4）：
  1. **前向执行**：按拓扑序执行图，缓存每个节点的输入和输出。
  2. **目标传播（反向）**：从输出节点开始，将真实输出（reference）设为输出节点的目标。然后沿反向拓扑序，对每个节点v，通过LLM引导的“后向推理”来推断其前驱节点u的目标输出。具体地，使用一个**后向推理提示模板**（图2），给定节点v的当前提示、原始输入、目标输出，要求LLM生成一个“目标输入” ˆI_v，使得当节点v处理这个目标输入时更可能产生目标输出。这个目标输入中隐含了前驱节点的目标输出。通过递归应用，每个子模块得到自己的局部目标输出 ˆO_u。
  3. **阶段式提示更新**：每个模块基于其局部更新集（包含输入、当前输出、目标输出）来更新其提示。提示被组织为有序的文本“声明”（claims）列表。更新过程由LLM作为优化器执行，使用提示优化模板（图3）。更新规则：添加新声明（每m+次迭代添加一个）、修改现有声明，但总体编辑预算（可修改的声明数）递减，使更新逐渐稳定。

### 关键技术细节

- **后向目标推断**：使用LLM来近似求解（5）式中的优化问题，即找到目标输入使得模块输出接近目标输出，同时保持与原输入相近（通过正则化系数λ）。公式（6）给出具体实现：`ˆIv = LLM(P_back(ˆOv, Iv, cv))`。
- **阶段式调度**：初始编辑预算n0，每m+次迭代允许添加一个新声明，每m-次迭代减少一个可编辑声明（不低于1）。这使提示长度增加但更新幅度减小，保证收敛。
- **理论分析**（基于简化的单token、单transformer块假设）：
  - **引理1**：提示更换可等效为权重更新（隐含权重定理）。
  - **定理1**：在提示长度稳定性假设下，隐含权重更新量级为O(t^{-1})，即随迭代递减。
  - **定理2**：局部目标损失的变化也以O(t^{-1})上界，即更新稳定。
  - **定理3**：若LLM优化器足够强（提示编辑距离小），整体损失与最优解的差距为O(t^{-1})，即能降低目标。
