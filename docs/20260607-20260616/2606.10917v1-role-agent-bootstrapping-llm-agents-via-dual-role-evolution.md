## 摘要
尽管大语言模型（LLM）智能体在复杂任务上展示了强大的性能，但它们的学习常常受到低效的交互反馈和静态训练环境的限制，从而阻碍了更广泛的泛化。为了解决这些限制，本文引入了Role-Agent，	extcolor{black}{一个框架}，它利用单个LLM同时充当智能体和环境，实现了一种引导式的协同演化。Role-Agent由两个协同组件组成：世界在智能体中（WIA）和智能体在世界中（AIW）。在WIA中，LLM作为智能体，在每个动作后预测未来状态；预测状态与实际状态之间的一致性被用作过程奖励，鼓励环境感知推理。在AIW中，LLM分析失败轨迹中的失败模式，并检索具有相似失败模式的任务，从而重塑训练数据分布以实现针对性练习。在多个基准测试上的实验表明，Role-Agent一致地提高了性能，相比强基线平均提升超过4%。

## Abstract
Although Large Language Model (LLM) agents have demonstrated strong performance on complex tasks, their learning is often limited by inefficient interaction feedback and static training environments, which hinder broader generalization. To address these limitations, this paper introduces Role-Agent, \textcolor{black}{a framework} that harnesses a single LLM to function concurrently as both the agent and the environment, enabling a bootstrapped co-evolution. Role-Agent comprises two synergistic components: World-In-Agent (WIA) and Agent-In-World (AIW). In WIA, the LLM acts as the agent and predicts future states after each action; the alignment between predicted and actual states is then used as a process reward, encouraging environment-aware reasoning. In AIW, the LLM analyzes failure modes from failed trajectories and retrieves tasks with similar failure patterns, thereby reshaping the training data distribution for targeted practice. Experiments on multiple benchmarks show that Role-Agent consistently improves performance, yielding an average gain of over 4\% over strong baselines.

## 2. 方法论：核心思想、关键技术细节、算法流程

- **核心思想**：利用一个LLM作为**双角色实体**，在训练过程中交替扮演**智能体**（负责行动与推理）和**环境**（提供反馈并调整任务分布），形成闭环协同进化。
- **关键技术细节**：
    1. **World-In-Agent (WIA)**：
        - 在每次交互的每个步骤，智能体在执行动作`a_t`后，被提示**预测未来H步的状态**`\hat{s}_{t,h}`（h=1,…,H）。
        - 使用**最长匹配子序列（LMS）** 计算预测状态与实际状态的相似度，得到**预测奖励矩阵**`\tilde{r}_{t,h}`∈[0,1]。
        - 将**任务奖励**`R_task`（从t步开始的折扣回报）与**预测奖励`R_pre`** 结合：`R_t = R_task(a_t) * (1 + R_pre(a_t))`。预测奖励作为**可靠性调制因子**：准确预测保留甚至放大任务奖励，不准确预测则削弱奖励，避免偶然成功的动作获得过高信用。
        - 进一步采用**状态分组**（基于文本状态哈希映射，相似度>0.9），计算**状态级优势**`A_S`，并与轨迹级优势`A_E`线性组合：`A = A_S + α·A_E`。最终用GRPO-style目标函数更新策略。
    2. **Agent-In-World (AIW)**：
        - 对于**失败轨迹**，由同一LLM（以环境角色）进行分析，生成**失败模式**（如实体混淆、位置错误等），包含类型、核心教训、检索查询。
        - 构建**离线失败模式库**，检索与当前失败模式**相似的历史任务**，将这些任务重新纳入训练集，**调整数据分布**，使智能体集中练习其薄弱环节。
    3. **算法流程**（文字描述）：
        - 初始化策略π_θ、参考策略π_ref、任务池D、失败记忆库M为空。
        - 每次迭代：
            - 从D采样一批任务。
            - 对每个任务，执行智能体，记录轨迹τ。
            - 在每一时间步，让智能体预测未来状态，计算预测奖励，调制任务奖励。
            - 使用状态分组计算状态级优势，结合轨迹级优势更新π_θ。
            - 对失败轨迹，由LLM分析失败模式并存入M。
            - 从M检索与当前失败模式相似的任务，加入D并更新采样分布。

## 6. 主要结论与发现

- Role-Agent在**所有基准上一致优于强基线**，平均提升超过4%（ALFWorld上相对GiGPO提升4.2%，WebShop上提升6.9%，搜索QA提升3.7%）。
- **WIA和AIW各自贡献独立且互补**，移除任一组件都会导致性能下降，但均仍优于GiGPO。
- **AIW通过动态调整训练分布**，使智能体在训练后期获得更快收敛和更高性能上限，同时减少了训练与推理的rollout不一致。

- 结论：双角色自举协同进化能有效增强LLM智能体的环境感知、泛化和多步推理能力。

## 7. 优点

- **创新性**：首次提出利用**单个LLM同时充当智能体与环境**实现自举协同进化，无需额外模型或人工监督，区别于只优化智能体端的自进化方法。
- **模块设计合理**：WIA将未来状态预测作为过程奖励，巧妙调制任务奖励，鼓励环境感知推理；AIW通过失败模式分析实现针对性的数据分布重塑，形成闭环。
- **实验充分且结果稳健**：在多个基准、多种模型规模、多次运行下均显示一致提升，消融与敏感性分析验证各组件有效性。
- **效率较高**：额外计算开销仅5.2%，几乎不增加训练成本。
- **可解释性好**：通过案例分析清晰展示了失败模式提取与任务检索的流程，便于理解。

## 8. 不足与局限

- **环境LLM的依赖**：AIW中若使用更强但不同主干的LLM（如GPT-4o）可能提升效果，但会引入额外知识，与同主干基线比较不公平；本文保持全实验使用相同主干，但也限制了AIW的分析能力上限。
- **状态分组阈值固定**：采用从GiGPO继承的0.9相似度阈值，可能限制跨任务泛化，未对分组策略进行自适应优化。
- **评估环境单一**：仅测试了**基于文本的交互环境**（ALFWorld、WebShop、搜索QA），未涉及多模态或实时具身场景；扩展到视觉或物理环境需结合视觉状态描述或潜在状态匹配。
- **失败模式库规模**：在ALFWorld训练中仅积累11种独特模式，虽然足以说明效果，但在更开放或更复杂的场景下，模式库的覆盖程度和检索效率可能需要进一步验证。
- **超参数敏感**：预测步数H超过5%·T_max后性能显著下降，说明远距离预测容易导致奖励投机或注意力稀释，需谨慎设定。
