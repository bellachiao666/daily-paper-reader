## 摘要
技能文档，即指导大型语言模型（LLM）Agent的结构化自然语言指令，对现代Agent框架至关重要，然而LLM很难撰写切实有效的技能。在SkillsBench上，人类撰写的技能将通过率提高了16.2个百分点，而LLM撰写的技能未带来可衡量的提升。我们提出SkillAxe，一个完全无监督的框架，使LLM能够迭代地诊断和精炼自身技能。SkillAxe将技能质量分解为四个可解释的维度（质量影响、触发精确性、指令符合性与故障归因、解决方案路径覆盖率），生成结构化改进简报，无需真实标签、测试套件或环境奖励。在SkillsBench上，SkillAxe将未改进的LLM技能的通过率相对提高了28%，并缩小了与人类撰写技能差距的47-67%。我们在SpreadsheetBench上验证了该方法作为持续改进引擎的效果，其中SkillAxe构建的技能库从过去的Agent轨迹中学习，仅用22个技能就将通过率从16.0%提升至52.0%。

## Abstract
Skill documents, structured natural-language instructions that guide Large Language Model (LLM) agents, are critical to modern agent frameworks, yet LLMs struggle to write skills that actually work. On SkillsBench, human-authored skills improve pass rates by 16.2 percentage points, while LLM-authored skills provide no measurable gain. We introduce SkillAxe, a fully unsupervised framework that enables LLMs to iteratively diagnose and refine their own skills. SkillAxe decomposes skill quality into four interpretable dimensions (quality impact, trigger precision, instruction compliance with fault attribution, and solution-path coverage), producing structured improvement briefs that require no ground-truth labels, test suites, or environment rewards. On SkillsBench, SkillAxe improves pass rates by 28\% relative over unimproved LLM skills and closes 47--67\% of the gap to human-authored skills. We validate the approach as a continuous improvement engine in the wild on SpreadsheetBench, where a SkillAxe-built skill library learns from past agent trajectories and raises pass rate from 16.0\% to 52.0\% using only 22 skills.

- **整体含义**：技能文档是指导LLM Agent达成任务的关键结构化自然语言指令，但其编写和精炼过程目前依赖人工、经验式和缺乏透明反馈。本文旨在提出一种无需人工监督、无需测试套件或环境奖励的自动化方法，使LLM能够自行诊断并迭代优化技能质量，从而缩小LLM技能与人类技能之间的差距。

## 2. 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：通过评估引导的自我精炼（Evaluation-Guided Self-Refinement），将技能质量分解为四个可解释的维度，生成结构化改进简报，再由LLM根据简报重写技能，迭代直至收敛。
- **关键技术细节**：
  1. **质量影响（Quality Impact）**：将Agent在无技能与有技能下的输出进行配对比较，通过LLM判断偏好方向（+1, 0, -1）和改善幅度（0~1），得到综合分数∈[-1,1]。用于判断技能整体是否有帮助。
  2. **触发精确性（Trigger Precision）**：提取技能中的正面触发短语（应该激活的任务）和负面触发短语（不应激活的任务），用带句子上下文的嵌入表示，计算三个几何诊断指标：
     - CoverageBreadth（正面短语间的平均余弦距离，衡量触发范围宽度）
     - NegativeSpecificity（负面短语到最近正面短语的距离，衡量排除有效性）
     - BoundarySharpness（最小正负短语距离，衡量决策边界清晰度）
  3. **指令符合性与故障归因（Instruction Compliance with Fault Attribution）**：将技能分解为可评估的规则，对每条规则评估相关性、遵守度（adherence）、规则质量（quality）、技能过错（fault）。通过加权计算Compliance分数和SkillScore分数，区分技能指令问题与Agent执行问题。
  4. **解决方案路径覆盖率（Solution-Path Coverage）**：枚举任务可能的多个解决路径，测量技能文档中每个片段与各路径的嵌入相似度，取最大值平均，判断技能是否支持多种有效执行策略。
- **流程**：  
  ① Agent分别在无技能和有技能条件下执行任务；  
  ② 从行为差异中计算四个诊断指标；  
  ③ LLM根据结构化改进简报重写技能；  
  ④ 重复直至收敛（本文实现了一次迭代）。

## 6. 论文的主要结论与发现

- **SkillsBench结果**：SkillAxe将LLM自写技能的通过率相对提高28%（32.5% → 41.6%），缩小了与人类技能差距的47-67%（人类技能通过率51.9%）。主要提升来自覆盖率的改善（从49.4%提升至72.7%），而非答案质量（质量指标几乎不变）。

## 7. 优点：方法或实验设计上的亮点

- **完全无监督**：无需任何任务标签、测试套件或环境奖励，仅利用任务指令和Agent输出即可诊断和优化。
- **可解释性**：四个诊断维度各有明确语义和几何/逻辑公式，改进简报可追溯每个诊断的贡献，便于人工理解。
- **故障归因**：区分技能指令错误与Agent执行错误，避免将正确的规则因Agent执行差而被错误修改。
- **连续改进能力**：通过SpreadsheetBench的部署场景展示了SkillAxe可作为生产环境的持续改进引擎，从历史轨迹中学习并压缩技能库。
- **多维度验证**：在通用任务和电子表格领域均验证有效性，并采用原生验证器和公平评分两种评估方式，提高可信度。

## 8. 不足与局限

- **结构性错位不可检测**：技能可能内部自洽但整体策略根本错误，此时规则级诊断无法识别。
- **忽略技能间交互**：当前每个技能独立评估，未考虑多个技能同时激活时的冲突或互补效应。
- **LLM judge偏差**：质量影响和指令符合性依赖LLM判断，可能存在位置偏差或校准问题。
- **单次迭代**：仅进行一轮改进，人类技能仍有较大差距，更多迭代可能进一步缩小差距但未探索。
- **实验覆盖有限**：SpreadsheetBench测试集仅50个任务，且准确性上限受任务难度限制；SkillsBench的试验次数少导致置信区间宽；无消融实验。
- **触发嵌入方法可能敏感**：使用预训练嵌入模型（all-MiniLM-L6-v2），不同模型可能导致不同诊断结果。
- **评估歧义**：公平评分保守处理无输出文件的情况，可能低估真实性能；SpreadsheetBench的公式缓存问题虽已处理，但仍可能残留工具差异。
