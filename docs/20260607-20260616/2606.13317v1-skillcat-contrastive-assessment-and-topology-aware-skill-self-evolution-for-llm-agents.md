## 摘要
面向LLM智能体的技能自进化方法旨在将执行轨迹转化为可复用的技能文档，但当前流程通常每个任务仅从单条轨迹学习，在检查前合并候选技能补丁，并在推理前加载完整技能库。我们提出SkillCAT，一个无需训练的框架，将该流程分为三个阶段。对比因果提取（CCE）为每个任务采样多条轨迹，并比较同任务成功/失败对，以识别解释结果差异的证据。评估增强进化（AAE）在源任务克隆上回放每个候选补丁，并在分层技能补丁合并前仅保留那些改善或维持任务结果的补丁。拓扑感知任务执行（TTE）将进化后的技能编译为可路由的子技能拓扑，因此推理仅加载与任务相关的能力节点。我们在常见智能体基准上评估SkillCAT，包括SpreadsheetBench、WikiTableQuestions和DocVQA，并进一步测试跨模型和分布外泛化性能。在这些设置下，SkillCAT相比基线平均得分提升高达40.40%，展示了无需模型训练即可实现可靠的技能进化。

## Abstract
Skill self-evolution methods for LLM agents aim to turn execution trajectories into reusable skill documents, but current pipelines typically learn from one trajectory per task, merge candidate skill patches before checking them, and load the full skill corpus before inference. We propose SkillCAT, a training-free framework that separates this process into three stages. Contrastive Causal Extraction (CCE) samples multiple trajectories for each task and compares same-task success/failure pairs to identify evidence that explains outcome differences. Assessment-Augmented Evolution (AAE) replays each candidate patch on source-task clones and keeps only patches that improve or preserve task outcomes before hierarchical skill patch merging. Topology-Aware Task Execution (TTE) compiles the evolved skills into a routable sub-skill topology, so inference loads only the capability nodes relevant to the task. We evaluate SkillCAT on common agent benchmarks, including SpreadsheetBench, WikiTableQuestions, and DocVQA, and further test cross-model and out-of-distribution generalization. Across these settings, SkillCAT raises the average score over baselines by up to 40.40%, demonstrating reliable skill evolution without model training.

## 1. 论文的核心问题与整体含义（研究动机和背景）

- **背景**：大语言模型（LLM）智能体在执行任务时通常依赖外部技能文档（如结构化指令、工具使用规则、任务经验等）。早期技能库由人工编写，近期研究从智能体执行轨迹中自动提取和精炼技能，即“技能自进化”（skill self-evolution），旨在将轨迹转化为可复用的技能文档，从而提升智能体在未见任务上的表现，且无需更新模型参数。
- **现有方法局限**：以 Trace2Skill 为代表的免训练技能自进化方法虽有效，但仍存在三个关键问题：
  1. **单轨迹偏差（Single-Trace Bias）**：每个任务仅从一条轨迹中学习，成功轨迹可能依赖偶然策略，失败轨迹难以定位根本原因。
  2. **未验证合并（Unvalidated Merging）**：从各轨迹提取的候选补丁不经独立验证就直接合并，低质量或有害补丁可能进入技能库。
  3. **上下文过载（Context Overload）**：合并后的技能库持续增长，推理时加载全部规则会增加提示长度、引入无关内容，干扰智能体决策。
- **核心目标**：提出一个无需训练、可分离的框架，同时解决上述三个问题，实现可靠、高效的技能自进化。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 2.1 整体框架：SkillCAT
SkillCAT 将技能生命周期分解为三个可观测阶段，全部在推理时以零训练方式执行。

- **阶段一：对比因果提取（Contrastive Causal Extraction, CCE）**
  - **核心思想**：利用同任务多种子采样的成功/失败轨迹形成对比对，提取能解释结果差异的因果证据，避免单轨迹偏差。
  - **技术细节**：
    - 对每个任务 xi，采用 K 个随机种子执行，获得轨迹集合 Ti = {(τ_i,z, y_i,z)}，y ∈ {0,1}。
    - 若任务同时含有成功和失败轨迹，则从 Ti⁺ 和 Ti⁻ 中各随机采样一条，构成对比对 (τ⁺, τ⁻)。
    - **因果分水岭（causal watershed）**：定位两条轨迹行动序列首次出现差异的时间步 w_i = min{t : α⁺_t ≠ α⁻_t}。
    - 提取器 E 围绕该分水岭生成候选经验记录 r_i = E(τ⁺, τ⁻, w_i)，包含局部证据、失败原因和可编辑的技能教训。
    - 若该任务所有轨迹全成功或全失败（无严格对比对），则可启用回退机制（单轨迹提取器 E1）或跳过该任务。

- **阶段二：评估增强进化（Assessment-Augmented Evolution, AAE）**
  - **核心思想**：将每个候选补丁视为待验证的假设，在源任务克隆上回放并打分，只保留能改善或保持任务结果的补丁，再进行分层合并。
  - **技术细节**：
    - 对候选补丁 p_j，在关联的源任务克隆集 X_j 上回放：记录基线结果 y_j 和回放结果 ȳ_j。
    - **评估得分**：根据结果转移赋予离散分数 a_j：
      - (0→1) 失败→成功：3.0（最强证据）
      - (1→1) 成功→成功：2.0（行为保持）
      - (0→0) 失败→失败：1.0（低权重）
      - (1→0) 成功→失败：0.0（拒绝）
    - 保留满足阈值 θ（默认 2.0）的补丁：P_θ = {p_j : a_j ≥ θ}。
    - **分层分级合并**：按得分层级从低到高依次合并，高得分补丁在后续阶段享有更高优先级。合并算子复用 Trace2Skill 的 Map-Reduce 风格编辑流程。

- **阶段三：拓扑感知任务执行（Topology-Aware Task Execution, TTE）**
  - **核心思想**：将进化后的技能库编译为可路由的子技能拓扑，推理时仅加载与当前任务相关的最小技能子集，避免上下文过载。
  - **技术细节**：
    - **拓扑构建**：离线解析 AAE 输出技能 S* 的 Markdown 层级：一级内容作为常驻核心技能 S°；二级章节作为可路由节点集合 V。每个节点 v 保留正文 B_v，并提取标题、关键词、摘要和依赖关系作为路由元数据 Σ。
    - **LLM 路由与技能装配**：给定任务描述 c 和路由预算 k，LLM 路由器 ρ 从 Σ 中选出最多 k 个相关节点。运行时执行确定性扩展（Expand），包括节点 ID 验证、依赖边扩展、基础节点注入，获得最终节点集合 V_c。
    - 路由技能：S_c = S° ⊕ ⊕_{v∈V_c} B_v。TTE 不重新生成技能文本，仅改变加载的规则集合。

## 6. 论文的主要结论与发现

1. **SkillCAT 显著超越现有方法**：在 SpreadsheetBench-Verified 上，Qwen3.5-35B-A3B 深度进化设定下 Vrf 达到 55.50%，比 Trace2Skill+Combined（29.67%）提升 25.83 个百分点；平均得分提升高达 40.40%（绝对）。
2. **各模块不可或缺**：消融显示去除任意模块均导致性能下降，其中 AAE 最关键（w/o AAE 后低于 Trace2Skill），CCE 次之，TTE 提供额外收益但单独使用不足。
3. **对比因果提取有效**：CCE 从同任务成功/失败对中提取的证据，比单轨迹提取更可靠，5 个种子是最优精度-成本平衡点。
4. **源任务回放是校准过滤器**：AAE 评分与下游性能严格单调相关：失败→成功补丁效果最好（+21.33pp），成功→失败补丁效果最差（-6.17pp），证实了验证的必要性。
5. **拓扑路由可大幅减少上下文**：LLM 图路由器在 top_k=7 时达到 55.50% Vrf，同时减少 41.6% 的上下文长度。
6. **技能可跨模型复用**：Qwen3.5-122B-A10B 进化的技能在 gemma-4-31B-it 上比 Human-Written 提升 31.50 个百分点 Vrf，在 gpt-5.4-mini 上也有提升。
7. **未验证合并可能有害**：Trace2Skill+Error 在 DocVQA 上低于 No Skill 基线，而 SkillCAT 避免了此问题。

## 7. 优点

- **方法创新**：将对比学习思想引入轨迹证据提取（CCE），通过同任务多种子对比精准定位因果分水岭，克服单轨迹偏差；首次在技能进化中引入源任务回放验证（AAE），用结果转移的离散得分实现可靠的补丁筛选；拓扑感知执行（TTE）将技能库组织为可路由图，实现运行时上下文精简。
- **无需训练**：整个框架基于推理和文本操作，无需梯度更新或微调，适用于任意 LLM，易于部署和扩展。
- **实验严谨**：涵盖多个领域（表格、表格查询、文档问答）、多种模型规模（35B、122B 以及跨族模型）、两种初始化设定（深度进化和创建式）；消融实验设计完备（单模块移除、仅使用单模块、超参数扫描）。
- **分析深入**：对种子数、评分校准、路由选择等关键设计维度提供了定量分析，验证了设计选择的合理性。
- **跨模型泛化验证**：展示了技能作者与用户可解耦，技能文档具有跨模型可迁移性，增强了方法的实用价值。

## 8. 不足与局限

- **场景覆盖不足**：实验仅聚焦于电子表格和文档问答，未在编程、网页操作、机器人控制等更复杂或动态的智能体环境中测试，通用性待进一步验证。
- **评估粒度的局限**：AAE 回放打分仅基于二元结果（成功/失败），未考虑部分成功或细粒度奖励，可能丢失部分有用信息；对于多步骤任务，回放覆盖度（是否跑完所有子任务）未详细说明。
- **种子数选择依赖经验**：5 个种子虽在实验中表现最佳，但未解释种子数增多后性能下降的根本原因（可能是采样噪声或轨迹多样性不足），缺乏理论指导。
- **资源消耗**：虽然免训练，但 CCE 阶段对每个任务需多次推理（种子数×任务数），AAE 阶段需对每个候选补丁进行源任务回放（可能数十次推理），总推理开销不小，对长尾大规模任务可能受限。
- **路由预算 k 的设定**：当前使用固定 top_k=7，未见对不同任务复杂度自适应调整的策略，且图路由依赖 LLM 元数据，对小模型可能不鲁棒。
- **潜在偏差风险**：CCE 对比对采样仅从成功/失败集合中随机选一条，若存在多种失败模式或多种成功路径，单次对比可能遗漏重要差异。
- **未讨论错误累积**：技能进化是一个多步骤过程，CCE→AAE→TTE 各阶段可能引入错误，论文未分析误差传播或鲁棒性边界。
- **缺乏与其他训练类方法对比**：论文仅比较了免训练方法（Trace2Skill），未与需要微调或端到端 RL 的技能进化方法（如 SKILL1、EvoSkill 等）对比，无法全面评估精度-开销权衡。
