## 摘要
在本文中，我们提出EEVEE，这是首个面向LLM代理的多数据集测试时提示学习框架，能够在现实任务流中进行测试时提示学习。现有方法主要针对单数据集场景设计，而现实应用要求模型处理来自多个数据集、领域和任务分布的异构输入流，限制了其实用性。为了减轻跨数据集干扰，EEVEE引入了一个路由模块，将输入划分为任务簇并分配到合适的提示配置中。该设计通过路由-提示协同进化策略进行优化，采用交错的路由学习和提示学习阶段来解决彼此的依赖关系。跨多个数据集的实验表明，该框架在异构数据流下提高了鲁棒性，同时保持了单基准学习能力和效率。具体而言，EEVEE在Qwen3-4B-Instruct和DeepSeek-V3.2上的平均多基准分数分别提高了10.38和24.32分，相比SOTA方法GEPA和ACE最高提升了37.2%和48.2%.

## Abstract
In this paper, we propose EEVEE, the first multi-dataset test-time prompt learning framework for LLM agents, enabling test-time prompt learning under real-world task streams. Existing methods are largely designed for single-dataset settings, while real-world applications require models to handle heterogeneous input streams drawn from multiple datasets, domains, and task distributions, limiting their practical applicability. To mitigate cross-dataset interference, EEVEE introduces a router that partitions incoming inputs into task clusters and assigns them to suitable prompt configurations. This design is optimized via a router-prompt co-evolution strategy, which employs interleaved router and prompt learning phases to address their mutual dependency. Experiments across multiple datasets demonstrate that the framework improves robustness under heterogeneous data streams while maintaining single-benchmark learning capability and efficiency. Specifically, EEVEE improves average multi-benchmark scores by 10.38 and 24.32 points over Qwen3-4B-Instruct and DeepSeek-V3.2, surpassing SOTA methods GEPA and ACE by up to 37.2% and 48.2%.

# EEVEE：面向现实世界中自我改进代理的测试时提示学习

## 1. 核心问题与整体含义（研究动机和背景）

- **问题背景**：现有测试时提示学习方法（如GEPA、ACE）大多适用于单数据集/单基准场景，即在同质、静态的输入流上学习一个共享提示。然而，真实世界部署中的输入流往往来自多个异构领域、任务格式和能力组合，形成“多数据集测试时提示学习”场景。
- **核心挑战**：单数据集方法在多任务混合流中会产生“跨数据集干扰”——针对某一领域更新提示会损害模型在另一个领域上的性能。实验表明，随着任务数量增加，GEPA和ACE的累积保留收益迅速变为负值。
- **研究动机**：设计一个能处理异构任务流的测试时提示学习框架，同时保持提示学习的轻量和自适应特性，使LLM代理能够在线自我改进。

## 2. 方法论

- **核心思想**：引入一个**路由器（Router）**，在推理时将输入动态划分为任务簇，并为每个簇分配一个**专用提示**。通过路由器-提示协同进化（Router-Prompt Co-evolution）策略，使路由决策和提示更新互相促进，而不是独立优化。
- **技术细节**：
  - **框架结构**：维护一个提示集 `P = {p₁,...,p_K}` 和路由器 `R`。推理时，路由器先选择提示槽 `z = R(x; P)`，然后模型用所选提示生成回答。
  - **路由器进化（Router Evolution）**：在固定提示集下，从临时路由器池中采样、变异、反射生成候选路由器，使用下游准确率、一致性和平衡性构成的分数 `S_R(R)` 进行评估，并在验证集上保留优于基线的新路由器。
  - **提示进化（Prompt Evolution）**：在固定路由器下，将数据按路由结果分组，对每个非空槽独立进化。采用帕累托前沿池（Pareto-frontier pool）保存互补提示，通过“空提示”阈值过滤无效编辑。
  - **三阶段训练过程**：
    1. **初始化（Initialization）**：在混合训练集上运行提示学习，从帕累托前沿池中贪婪选择覆盖互补样本的Top-K提示，形成初始提示集，确保路由器有区分能力的基础。
    2. **探索（Exploration）**：交替进行路由器和提示进化（轻量预算），使用退火机制从平衡性权重逐渐转向准确率权重，保持早期多样性、后期稳定性。
    3. **收敛（Convergence）**：当路由器稳定后固定之，分配更大预算给每个提示槽进行精细化学习。
- **关键创新**：路由器-提示协同进化解决了二者的相互依赖问题：路由决定每个提示的输入分布，提示质量又决定路由是否有效。

## 6. 主要结论与发现

- **性能提升显著**：EEVEE在Qwen3-4B上平均得分51.75，比基线高10.38；在DeepSeek-V3.2上平均64.07，比基线高24.32，并超越GEPA和ACE最高48.2%。
- **多任务保留优势明显**：四个任务逐步加入时，EEVEE累积保留收益始终为正（最终+41.53），而GEPA和ACE降为负值（-15.36、-18.58）。
- **单任务仍具竞争力**：在单基准设置下，EEVEE与GEPA、ACE相当甚至更优（Formula:55.25, HumanEval:73.17）。
- **跨模型泛化良好**：用Qwen3学得的提示直接用于DeepSeek-V3.2，平均提升14.35分。
- **跨任务泛化有限**：在类似领域（MBPP）有微弱提升（+1.13），在无关领域（MMLU-Pro）略有下降（-1.82），说明提示学习偏向任务特定程序而非通用知识。
- **案例总结**：提示学习擅长捕获可复用的过程性策略（如代码格式、公式计算规则），但在需要领域知识的闭卷问答中可能因过度形式化推理而忽略事实，导致负向切换。

## 7. 优点

- **创新性**：首个面向多数据集异构流的测试时提示学习框架，解决了关键的现实问题。
- **方法优雅**：路由器+多提示的设计直观且有效；三阶段训练协同进化解决耦合问题。
- **实验全面**：覆盖多任务、单任务、泛化、消融、成本、案例和超参数鲁棒性，对比方法先进。
- **效率较高**：路由设计仅增加少量令牌开销，远低于ACE的提示膨胀。
- **可解释性**：通过案例和负向案例分析了提示学习的能力边界，揭示了“过程性 vs. 知识性”的权衡。

## 8. 不足与局限

- **随机性**：基于LLM的进化过程无法保证精确复现，多次运行可能产生不同路由器和提示集。
- **依赖标签**：仍需有标注的适应集（或规则标签）来驱动反馈，不是完全基于反思的学习器。
- **在线性有限**：目前使用预划分的训练/验证集，而非完全在线流，实际部署仍需准备高质量适应数据。
- **跨任务泛化能力弱**：在知识密集型（MMLU-Pro）反而轻微下降，可能限制部分应用。
- **实验范围限制**：仅在两个中等规模模型（4B和V3.2）上验证，未在更大模型（如70B+）或更多领域（如多模态、对话）上测试。
- **社会风险**：噪声或错配的适应数据可能导致模型强化错误启发式，需谨慎验证后再部署。
