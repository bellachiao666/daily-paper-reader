## 摘要
大型语言模型（LLM）智能体依赖可复用的技能来解决复杂任务。然而，现有的技能创建方法将技能视为孤立且静态的工件，限制了其可复用性、可靠性和长期改进。我们提出MUSE-Autoskill智能体（利用记忆的技能进化），这是一种以技能为中心的智能体框架，通过在一个统一的生命周期（创建、记忆、管理、评估和改进）中创建、复用和优化技能，使智能体能够持续提升其任务解决能力。我们的框架使智能体能够按需创建技能，跨任务存储和复用技能，高效地组织和选择技能，并通过单元测试和运行时反馈对技能进行评估以持续改进。我们进一步引入了技能级记忆，为每个技能累积跨任务的体验，从而实现更有效的复用和随时间推移的适应。在SkillsBench上的实验提供了初步证据，表明生命周期管理的技能能够提高任务成功率、效率、复用率以及跨智能体迁移能力，突显了将技能视为长期存在、经验感知且可测试资产的重要性。

## Abstract
Large language model (LLM) agents rely on reusable skills to solve complex tasks. However, existing skill creation approaches treat skills as isolated and static artifacts, limiting their reusability, reliability, and long-term improvement. We propose MUSE-Autoskill Agent (Memory-Utilizing Skill Evolution), a skill-centric agent framework that lets agents continuously improve their task-solving capability by creating, reusing, and refining skills under a unified lifecycle (creation, memory, management, evaluation, and refinement). Our framework enables agents to create skills on demand, store and reuse them across tasks, organize and select them efficiently, and evaluate them through unit tests and runtime feedback for continuous refinement. We further introduce skill-level memory that accumulates experience for each skill across tasks, enabling more effective reuse and adaptation over time. Experiments on SkillsBench provide initial evidence that lifecycle-managed skills can improve task success, efficiency, reuse, and cross-agent transfer, highlighting the importance of treating skills as long-lived, experience-aware, and testable assets.

## 2. 方法论：核心思想、关键技术细节

### 2.1 核心思想
*   **MUSE-Autoskill 智能体**（Memory-Utilizing Skill Evolution）：以技能为中心，将技能视为“一等公民”，围绕其生命周期构建完整的智能体系统。
*   **技能包**：遵循 Anthropic Agent Skills 格式，每个技能是一个目录，包含 `SKILL.md`（接口定义）、`scripts/`（可执行代码）、`tests/`（单元测试）、`resources/`（辅助数据）。
*   **五阶段生命周期**：
    1.  **创建（Creation）**：通过内置 `skill_create` 工具按需生成，在 ReAct 循环内调用，消除创建-使用脱节。生成后先运行单元测试，通过后才注册到技能库。
    2.  **记忆（Memory）**：多级记忆体系：
        *   **技能级记忆**（独特创新）：每个技能有一个 `.memory.md` 文件，随任务运行追加经验、失败模式、输入注意事项等，下次使用时自动加载。
        *   **短期记忆**：当前任务的上下文，可自适应压缩。
        *   **长期记忆**：跨会话的通用经验（如“偏好批量 I/O”），不被压缩。
    3.  **管理（Management）**：技能库通过元数据索引，采用渐进式披露（仅将目录注入系统提示）。支持合并（冗余技能）、修剪（失效或无用技能）、更新。
    4.  **评估（Evaluation）**：运行技能包内的单元测试，以及运行时执行反馈。测试失败触发自动精炼。
    5.  **精炼（Refinement）**：根据测试失败错误信息自动调用 `update_skill` 进行补丁或重生成，重新测试直到通过。

### 2.2 关键技术细节
*   **上下文管理**：基于 DAG 的会话节点，保持完整历史（不可变指针）和活动链（可变父指针）。两级压缩：
    *   L1（单节点压缩）：当某节点（如大工具输出）超过阈值（15K tokens）时，将其摘要替换。
    *   L2（链压缩）：当 L1 后仍超预算（180K tokens），将中间一段连续节点合并为一个摘要节点。
*   **技能执行**：通过统一的沙箱工具（`create_sandbox`, `sandbox_run` 等）在隔离进程中执行，失败、副作用仅限单次技能调用。
*   **跨会话持久化**：每次会话结束保存完整状态（DAG、技能使用记录等），支持从中间状态恢复。
*   **跨智能体迁移**：技能包以纯文本形式存在，不绑定特定智能体框架，可直接注入其他智能体（论文实验验证了 Hermes）。

## 6. 主要结论与发现

1.  **手工技能显著提升所有智能体**：MUSE 在手工技能帮助下达到 68.40%（+15.21 pp），高于 Codex（67.28%）和 Hermes（61.21%）。

3.  **自生成技能可高效迁移**：注入 Hermes 后，准确率从 47.89% 升至 58.40%（+10.51 pp），缩小了与 Hermes 使用手工技能的差距 79%；使用相同生成技能时，Hermes（58.40%）与 MUSE（60.35%）仅差约 2 pp，证明技能是纯粹的知识资产，不依赖特定智能体。

## 7. 优点

*   **统一生命周期**：首次将创建、记忆、管理、评估、精炼五个阶段集成在一个训练免费框架内，覆盖了现有方法（如 Voyager、AutoSkill、SkillGen）的缺失环节。
*   **技能级记忆**：独特的多级记忆，使技能能在不同任务间积累和使用经验，类似“技能的经验日志”。
*   **测试驱动评估与自动精炼**：单元测试不仅用于创建验证，失败后自动触发 refinement，形成闭环，提高可靠性。
*   **自适应上下文压缩**：DAG 结构 + 两级压缩，有效处理长任务历史，不掉失信息，支持跨会话恢复。
*   **跨智能体迁移验证**：明确设计了迁移实验，证明技能是去中心化的知识制品，不绑定特定代理。
*   **实际部署证据**：已在 ByteDance 生产系统中的 SkillMarket、ArkClaw、SkillHub 等平台应用，表明生命周期抽象具有实用性。

## 8. 不足与局限

*   **实验覆盖不完整**：只评估了 51/94 个 SkillsBench 任务，排除的任务环境更复杂、可能更难，导致报告数值可能高估实际性能。
*   **技能生成覆盖率低**：仅 68.6%（35/51）任务成功生成技能，底层瓶颈是 Phase 1 的无技能成功率，而非生成质量。失败任务集中在需要专用生产工具或密集数值推理的领域。
*   **单轨迹蒸馏风险**：每个技能仅从一条成功轨迹蒸馏，可能学习到特定于源轨迹的假设（如固定文件名、数值范围），跨任务泛化性有限（论文给出 hvac-control 回归案例）。
*   **迁移方向单一**：只验证了 MUSE → Hermes，未测试其他组合或反向迁移。
*   **统计不确定性**：每个任务仅 5 次运行，对二值奖励任务来说置信区间较宽。
*   **骨干模型局限**：仅使用 GPT-5.5，未验证其他模型（如开源模型）的适用性。
*   **自创建实验可能有评估偏差**：技能从同一任务的 Phase 1 轨迹生成，又在同一任务上评估，尽管验证器是确定性的且未硬编码答案，但仍可能存在过拟合路线。
