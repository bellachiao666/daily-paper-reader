## 摘要
自我进化的多智能体系统已成为LLM智能体从经验中持续改进的有前景的途径，其基础是持久记忆。然而，现有设计几乎全部采用跨智能体共享的集中式存储库，这增加了通信和协调开销，引发了隐私问题，并削弱了智能体的多样性。我们提出DecentMem，一种去中心化记忆框架，其中每个智能体维护自己的双池记忆——一个利用池（整合过去轨迹）和一个探索池（针对未见上下文由LLM生成的候选）。两个池基于来自LLM作为评判者的阶段性反馈进行在线重新加权。理论上，我们证明该设计保证了解决方案空间的全局可达性，并实现了O(log T)的累积遗憾，在常数范围内匹配随机赌博机下界。在实践中，跨越三个MAS框架（AutoGen、DyLAN、AgentNet）、三个Qwen3骨干网络（4B/8B/14B）、两个Gemma4骨干网络（E2B/E4B）以及涵盖数学、代码、问答和具身任务的五个基准测试，DecentMem相比最强的集中式记忆基线平均准确率提升高达23.8%，相比无记忆基线提升高达52.5%，同时令牌使用量减少高达49%。

## Abstract
Self-evolving multi-agent systems (MAS) have emerged as a promising route to LLM agents that continually improve from experience, with persistent memory at their foundation. However, existing designs almost exclusively adopt a centralized repository shared across agents, incurring communication and coordination overhead, raising privacy concerns, and collapsing agent diversity. We propose DecentMem, a decentralized memory framework in which each agent maintains its own dual-pool memory -- an exploitation pool of consolidated past trajectories and an exploration pool of LLM-generated candidates for unseen contexts. The two pools are reweighted online based on stage-wise feedback from an LLM-as-a-judge. Theoretically, we prove that this design guarantees global reachability of the solution space and achieves $O(\log T)$ cumulative regret, matching the stochastic bandit lower bound up to constants. In practice, across three MAS frameworks (AutoGen, DyLAN, AgentNet), three Qwen3 backbones (4B/8B/14B), two Gemma4 backbones (E2B/E4B) and five benchmarks spanning math, code, QA, and embodied tasks, DecentMem improves average accuracy by up to 23.8% over the strongest centralized memory baseline and by up to 52.5% over the no-memory baseline, while reducing token usage by up to 49%.

### 论文提出的方法论：核心思想、关键技术细节

- **核心思想**：DecentMem（Decentralized Memory）——**每个智能体独立维护私有双池记忆**，并基于LLM裁判的阶段性反馈在线调整两池权重，从而在利用历史经验与探索新策略之间实现自适应平衡，同时保留智能体多样性。

- **关键技术细节**：
    1. **双池记忆结构**（Dual-Pool Memory）：
        - **利用池（E-pool）**：存储来自过去成功任务的**整合轨迹**（包含动作序列、协作路径、自评注等），用于相似性检索驱动的局部搜索。
        - **探索池（X-pool）**：存储针对未见上下文由LLM**即时生成的候选记忆**，仅用于当前任务，维护新鲜的概率质量，避免陷入局部最优。
    2. **在线路由器（Online Router）**：
        - 在每个阶段，根据两池权重计算选择概率 \( \alpha_{E\text{-pool}} = w_E / (w_E + w_X) \)，按概率采样。
        - 选择E-pool后，执行相似度检索（需超过阈值τ）；若未匹配则回退到X-pool。
        - 选择X-pool则直接生成全新记忆。
    3. **阶段性更新**（Stage-wise Update）：
        - 执行完任务后，由**LLM裁判**对每个阶段的输出（正确性、分配质量、中间连贯性、最终整合性）逐阶段评分。
        - 定义差分信号 \( \Delta_t = I[\text{当前分} > \text{前一分}] \)。
        - 若E-pool被选中且得分上升：权重增加（加α=0.5）；否则衰减（乘以β=0.5，下界为1.0）。
        - 若X-pool被选中且得分上升：E-pool权重衰减（抑制过度利用）；否则增加（鼓励从失败中学习）。
        - 任务结束后，X-pool内容**合并**到E-pool，X-pool重置。
    4. **理论分析**：
        - 将搜索建模为带**启发式传送**的图随机游走：E-pool对应局部转移矩阵T，X-pool对应全支撑先验h。组合后转移矩阵严格正，保证Markov链不可约、非周期 → **全局可达性**（不会永久困于局部最优）。
        - 在线路由视为随机赌博机问题，证明在严格凹性假设下累积遗憾为**O(log T)**，匹配下界Ω(log T)，即**阶最优**。
