## 摘要
智能体强化学习（Agentic RL）使大语言模型智能体能够从环境奖励中持续改进，但所得策略并未系统性地积累可跨任务泛化的可复用策略。模块化技能可提供此类可复用策略，然而现有技能增强型强化学习方法将技能创建与策略优化解耦，可能导致采用与不断演化的策略相冲突的技能。受Anthropic的Skill Creator启发，我们提出ReSkill——一种将技能演化与策略学习相调和的强化学习循环内技能创建框架。ReSkill利用GRPO的组结构，仅以微小额外开销自然嵌入三种机制：（1）基于断言的技能创建器，从过往经验中诊断失败并提出条件性、触发式的技能修订；（2）组内轨迹采样，实现技能版本的可控比较，捕获最能支持策略持续学习的版本；（3）采用自适应折扣的汤普森采样，在策略演化过程中平衡技能版本选择的探索与利用。在多个领域中，ReSkill持续优于现有基于记忆和技能的强化学习方法，在未见任务上提升最为显著。技能生命周期分析显示，随着策略改进，技能被自动创建、测试、精炼和剪枝，证明了调和后的技能-策略共同演化。

## Abstract
Agentic reinforcement learning (RL) enables LLM agents to improve continuously from environment rewards, yet the resulting policies do not systematically accumulate reusable strategies that generalize across tasks. Modular skills can provide such reusable strategies, yet existing skill-augmented RL methods decouple skill creation from policy optimization, risking adopting skills that conflict with the evolving policy. Inspired by Anthropic's Skill Creator, we introduce ReSkill, an RL-in-the-loop skill creation framework that reconciles skill evolution with policy learning. ReSkill exploits the group-wise structure of GRPO to naturally embed three mechanisms with only marginal additional overhead: (1) an assertion-driven skill creator that diagnoses failures from past experience and proposes conditional, trigger-based skill revisions; (2) within-group rollout sampling that enables controlled comparison of skill versions, capturing which version best supports the policy's ongoing learning; and (3) Thompson Sampling with adaptive discounting to balance exploration and exploitation in skill version selection as the policy evolves. Across several domains, ReSkill consistently outperforms existing memory and skill-based RL methods, with the largest gains on unseen tasks. Analysis of the skill lifecycle shows skills being automatically created, tested, refined, and pruned as the policy improves, demonstrating reconciled skill-policy co-evolution.

## 2. 方法论：核心思想、关键技术细节

### 核心思想
- 受Anthropic Skill Creator启发，将人类参与循环替换为RL训练中的奖励信号，使技能创建与策略学习同步进行。
- 利用GRPO每组轨迹共享相同任务和策略的特点，在组内分配不同技能版本，实现可控比较，从而评估技能版本对策略的效果。

### 关键技术细节
1. **组内技能测试（Within-Group Skill Testing）**  
   - 在GRPO的每组`G`条轨迹中，每条轨迹根据Thompson Sampling分配一个技能版本（`S_new`或`S_old`），生成轨迹。  
   - 所有轨迹共享同一任务`q`和策略`π_θ`，唯一变量是技能版本，因此可以直接比较技能版本效果。  
   - 每条轨迹同时用于：(1) GRPO策略梯度更新；(2) 填充经验池供技能创建使用；(3) 更新Thompson Sampling后验。

2. **RL循环内技能创建（RL-in-the-Loop Skill Creation）**  
   - **经验池**：收集成功-失败轨迹对，持续刷新。  
   - **条件技能加载**：技能由`(content, trigger)`组成，只有触发条件满足时才加载到上下文。  
   - **断言驱动失败分级**：动态断言集`A`对轨迹进行规则检查，得到各断言通过率。LLM分析器结合通过率和轨迹样本，产生优先级排序的故障诊断，并更新断言集。  
   - **历史知情人技能修订**：基于当前技能库、故障诊断和先前提议的接受/拒绝历史，技能创建器生成候选版本`S_new`（包含ADD、MODIFY、DELETE操作）。触发条件在进入测试前进行充分性验证。

3. **自适应Thompson Sampling驱动的技能演化**  
   - 将新旧技能版本视为两只臂，成功概率服从Beta分布（初始`Beta(1,1)`）。  
   - 每个训练步骤，通过蒙特卡洛采样估计`Pr(p_new>p_old)`，并裁剪到`[ε_ts, 1-ε_ts]`作为选择新版本的概率，按此概率分配组内轨迹（公式(3)）。  
   - **自适应折扣后验更新**（公式(4)）：为应对策略漂移，每步先将旧伪计数乘以折扣权重`w_t=1/(1+n_t/M)`，再加入新观测。`M`为记忆参数，通过已完成测试周期的预测似然最大化自动估计（无需手动调参）。  
   - **接受/拒绝决策**：测试周期结束后，若折扣后验下`E[p_new] > E[p_old]`，则接受`S_new`为新基线；否则回退到`S_old`，立即开始下一轮测试。

## 3. 实验设计

### 数据集/场景
- **ALFWorld**：基于文本的室内环境，包含3类任务（Pick&Place、Transform、Examine），训练约3800个游戏，评估140个已见和134个未见配置。
- **Agentic Search**：智能体通过迭代搜索回答事实性问题。训练数据来自NQ和HotpotQA（各约3000问）。评估5个未见数据集：PopQA、TriviaQA、2WikiMHQA、MuSiQue、Bamboogle。

- **InterCode-SQL**：交互式SQL生成，训练1027个难度均衡的查询，评估1000个 held-out 例子。
- **WANDS**（网页购物）：基于产品搜索相关性，训练3000产品，评估496个产品（含2-opt和3-opt难度）。

- ReSkill在所有基准上一致优于现有记忆和技能增强RL方法，ALFWorld总体89.8%（4B）/92.7%（8B），Search总体47.6%/49.8%，分别比最强基线高出5.9%/6.9%和2.5%/2.3%。

## 7. 优点

- **方法创新**：首次将人类循环技能创建过程嵌入RL训练，实现技能-策略的调和共同演化，且不额外增加采样代价（利用已存在的GRPO组结构）。
- **自适应机制**：自适应Thompson采样结合策略漂移折扣，自动平衡新旧技能的探索与利用，并在线估计记忆参数`M`。
- **断言驱动诊断**：通过断言集对全经验池进行规则检查，比逐个轨迹LLM诊断更高效、更全面。
- **实验全面**：覆盖多个领域、两种模型规模，并提供丰富的消融和敏感性分析。
- **定性分析深入**：通过技能生命周期和案例研究展示了技能产生、优化、淘汰的动态过程，揭示学习机制。

## 8. 不足与局限

- **演化频率敏感**：在ALFWorld上，过低演化频率（`K_e=20`）导致性能下降，仍需每领域调参。作者建议未来工作探索自适应演化调度。
- **技能调用方式**：基于触发器无条件加载技能，模型对何时应用技能控制较弱。有时模型错误地将技能名称当作环境命令执行（“meta-bug”），或过度应用不合适的技能。作者建议未来探索工具调用形式，提供更精细的控制。
- **技能容量负担**：添加技能会增加上下文长度，增加模型认知负担。小模型易混淆技能指导与环境动作，或无法识别何时不适用技能。需通过格式感知奖励或其他训练手段改善。
- **未与最前沿训练算法结合**：ReSkill基于GRPO，但可与其他GRPO变体（如DAPO、Dr.GRPO）集成，文中未验证。
- **可解释性有限**：技能创建器依赖LLM（Claude或基础模型），其决策过程不完全透明，可能引入隐式偏差。
