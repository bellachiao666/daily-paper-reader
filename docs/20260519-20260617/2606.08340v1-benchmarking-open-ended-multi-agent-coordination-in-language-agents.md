## 摘要
随着语言模型越来越多地被部署为自主智能体，它们必须在开放式交互任务中长时间地与其他智能体协调。然而，现有的评估很少同时测试这些需求，而是侧重于单智能体任务、短交互或高度结构化的多智能体场景。我们提出了alem，一个基于JAX的开放式多智能体协调基准，构建在类似Craftax的动态之上。Alem将程序化生成的协调任务、软专业化、通信和可控制的协调难度嵌入到一个包含探索、制作、交易和战斗的长时生存世界中。我们在同质团队中零样本评估了13个现代LLM，并以训练过的MARL智能体作为参考点。当前的LLM智能体远未解决alem，平均仅约6%的归一化回报，但它们的失败并不均匀。在最困难的协调设置中，零样本的Gemini-3.1-Pro-High接近了训练了十亿步的MARL智能体，而GPT-5.4-High在基础任务奖励上表现强劲，但协调奖励却低得多。这种对比表明，个体任务能力并不等同于协调能力。消融实验显示，通信对协调贡献最大，而记忆和推理在用于维持多步骤计划时也有帮助。总体而言，我们的结果将协调确定为前沿LLM智能体区别于单智能体能力的一个独特瓶颈。Alem使这个瓶颈可测量，并为开发能够通信、分配角色和执行共享计划的智能体提供了受控测试平台。代码可在https://github.com/alem-world/alem-env获取。

## Abstract
As language models are increasingly deployed as autonomous agents, they must coordinate with others over long horizons in open-ended interactive tasks. Yet existing evaluations rarely test these demands together, instead emphasising single-agent tasks, short interactions, or highly structured multi-agent settings. We introduce $alem$, a JAX-based benchmark for open-ended multi-agent coordination built on Craftax-like dynamics. Alem embeds procedurally generated coordination tasks, soft specialisation, communication, and controllable coordination difficulty into a long-horizon survival world with exploration, crafting, trading, and combat. We evaluate $13$ modern LLMs zero-shot within homogeneous teams, with trained MARL agents as reference points. Current LLM agents remain far from solving alem, averaging only ~6% normalised return, but their failures are not uniform. On the hardest coordination setting, zero-shot Gemini-3.1-Pro-High approaches MARL agents trained for one billion steps, while GPT-5.4-High achieves strong base-task reward but much lower coordination reward. This contrast shows that individual task competence does not imply coordination competence. Ablations show that communication is the largest contributor to coordination, while memory and reasoning help when used to maintain multi-step plans. Overall, our results identify coordination as a distinct bottleneck for frontier LLM agents, separate from single-agent capabilities. Alem makes this bottleneck measurable and provides a controlled testbed for developing agents that communicate, allocate roles, and execute shared plans. Code is available at https://github.com/alem-world/alem-env.

## 1. 核心问题与整体含义（研究动机和背景）
- 当前语言模型（LLM）越来越多地作为自主智能体部署，需要在开放式、多智能体环境中长期协调，例如通信、分配角色、维护共享计划。
- 现有基准存在明显缺口：
  - 单智能体任务（如 BALROG）不测试协调。
  - 多智能体基准（如 MultiAgentBench、Collab-Overcooked）任务时间短、结构固定，缺乏长时程开放世界特性。
  - 多智能体强化学习（MARL）基准（如 SMAC、Overcooked）只覆盖协调谱的特定区域（同步或短窗口交接），且缺乏可控协调难度。
- 本文旨在填补这一空白，提出 **ALEM**，一个**长时程、开放式、具有程序化协调任务和可控难度**的多智能体基准，专门测试 LLM 智能体的协调能力。

## 2. 方法论：核心思想、关键技术细节
- **环境构建**：基于 Craftax-class 开放世界 (Craftax / Craftax-Coop)，在 JAX 中实现，支持快速并行模拟。
- **四方面核心扩展**：
  1. **多样协调类型**：同步（同时间步动作）、交接（短时间窗口内接力）、长期依赖（资源积累和跨步规划）。覆盖全面协调谱。
  2. **程序化协调生成**：每局动态采样协调任务（挖掘、建造、战斗、制作），类型（同步/交接）和参与智能体数量随机，避免模式记忆。
  3. **软专业化**：每个智能体有角色（觅食者/矿工/战士），非专业者执行角色动作有失败概率（`η_non = 1 - α`），专业者成功率为1。鼓励角色分配但允许在线适应。
  4. **显式通信通道**：广播离散消息，支持资源请求、队友定向给予、多步计划。
- **难度控制**：单一标量 `α ∈ [0,1]`（Easy=0.3, Medium=0.6, Hard=0.9）调节：
  - N-智能体要求的概率、交接窗口长度、软任务单人失败概率、非专业效率等。
  - 同一生成世界的空间布局和协调机会数量不变，仅调整执行约束，实现干净隔离。
- **接口**：提供像素、符号和**文本接口**（供 LLM 智能体使用）。文本接口包含系统提示、8 步历史、当前观测（地形、队友、协调目标、合法动作等），要求模型返回 `<action>`, `<communication>`, `<scratchpad>` 格式响应。
- **MARL 基线**：训练 IPPO、MAPPO、PQN-VDN、HyperMARL-IPPO，作为零样本 LLM 的参考点。

## 3. 实验设计
- **数据集/场景**：ALEM 本身包含 9 个程序化生成的关卡（地表、矿井、火焰界等），每局独立采样协调任务。评估在 Easy、Medium、Hard 三个难度级别进行。
- **基准方法**：
  - **LLM 智能体**（零样本，同质团队）：13 个现代模型，包括闭源 GPT-5.4-High、Gemini-3.1-Pro-High，开源 Gemma-4 系列、Llama-3.1/3.3、Qwen-3.5/3.6 系列（参数范围 8B 到 122B，含 MoE 变体）。
  - **MARL 基准**：训练 1 亿步（100M）和 10 亿步（1B）环境步的四个算法。
- **评估指标**：归一化回报（Base%、Coord%、Total%），各自独立归一化。同时报告协调奖励分解、合作事件计数等。
- **消融实验**（Hard 难度）：对最强模型 Gemini-3.1-Pro-High 和 Gemma-4-31B-it 分别移除通信、记忆（scratchpad）、降低/禁用推理。
- **异质团队实验**：同族（三个 Gemma 模型）和跨族（Gemini+GPT+Gemma）混合团队，比较总回报。

## 4. 资源与算力
- MARL 训练：使用 **NVIDIA L40S (48 GB)** 单 GPU，训练 10 亿步在约 **2 天**内完成。还使用 NVIDIA A100 80GB 用于 LLM 评估。
- LLM 评估：在 **3 台 NVIDIA A100 80GB** 设备上运行，每个模型每难度 20 种子（开源）或 10 种子（闭源），耗时 10–30 小时。

## 5. 实验数量与充分性
- **总实验量**：13 个 LLM × 3 难度 × 至少 10 种子 = >390 次完整评估；加上 MARL 基线（4 算法 × 3 难度 × 5 种子）、消融（2 模型 × 3 条件）、异质团队（2 设置）→ 实验组数充足。
- **公平性**：
  - LLM 零样本，统一提示模板，行动掩码，保守解析失败处理。
  - MARL 基线经过超参数扫描选择最优配置，报告 95% 置信区间。
  - 消融实验分别处理不同模型约束（Gemini 只能降低推理，Gemma 可完全禁用）。
- **充分性**：覆盖多种模型族、参数规模、训练计算量；诊断性分析（按任务类型分解、信息论诊断）验证了环境内在协调需求；定量和定性相结合。
- **局限性**：未包含 VLM、Claude 模型；闭源模型种子数较少（10）；异质团队仅初步实验。

## 6. 主要结论与发现
1. **ALEM 远未解**：LLM 平均 Total% 仅约 6%；最强 MARL (HyperMARL, 1B 步) 在 Hard 下也仅 15.3% Total%。
2. **个体任务能力强不等于协调能力强**：GPT-5.4-High 具有高 Base% 但 Coord% 很低；相反小模型如 Gemma-4-26B-A4B-it 协调相对较好。
3. **通信是协调的最大贡献因素**：移除通信导致 Gemini 协调 Coords% 从 17.5→5.3，Gemma 从 8.8→3.8；Base% 变化较小。
4. **记忆和推理的作用取决于使用方式**：Gemini 用 scratchpad 做前瞻计划（71.5% 多行、75.3% 含未来词），移除后 Coord% 下降；Gemma 用 scratchpad 做状态摘要，移除影响小。
5. **异质团队表现接近同质平均**：混合组合同质团队均线，未自动提升到最强成员水平。
6. **协调类型难度差异**：交接任务相对易解决；同步硬任务（同时间步+N 智能体）非常困难；建造任务（长程计划）几乎无 LLM 取得进展。

## 7. 优点
- **全面性**：同时覆盖长时程、开放式、多种协调类型、可控难度、通信、软专业化，填补重要空白。
- **程序化生成**：减少记忆和污染，增强泛化测试。
- **高效 JAX 实现**：支持大规模 MARL 训练（1B 步 <2 天），可复现。
- **细粒度诊断**：分解 Base%、Coord%、任务类型覆盖率，揭示不同失败模式。
- **可扩展**：提供像素、符号、文本接口，支持 MARL 和 LLM 研究。
- **开放公平**：代码开源，超参数扫描，统计严谨（95% bootstrap CI）。

## 8. 不足与局限
- 实验覆盖不足：未评估 VLM（视觉-语言模型）和 Claude 系列；闭源 API 成本限制了种子数（10）。
- 异质团队实验初步：只测试了简单混合策略，未深入分析通信协议对齐、规划视野差异。
- 评估仅限零样本：未尝试微调、上下文学习、记忆增强等更强方法。
- 环境本身仍有提升空间：例如添加动态任务、连续通信、更精细的语言理解。
- 可能残留偏差：提示工程影响 LLM 表现；模型输出解析失败率低但存在。
