## 摘要
随着大型语言模型（LLM）的广泛部署，它们越来越需要与其他AI智能体协同工作，而非孤立运行。在这些场景中，有效协调要求智能体在不确定性下进行通信、共享信息并做出决策。我们提出了SMAC-Talk，这是星际争霸多智能体挑战的一个自然语言扩展，用于评估基于LLM的智能体在协作多智能体环境中的表现。该环境具有分散控制、部分可观测性和长时域决策等关键特征。SMAC-Talk包含一个自然语言通信通道，用于探测智能体的协调与信任。我们利用该通信通道构建了不同的评估场景，包括嵌入一个试图仅通过通信来干扰和欺骗盟友的欺骗性通信者的设置。我们提供了三个用于基准测试的智能体，使用了Qwen3.5家族的四个模型，并研究了推理结构、记忆和模型规模如何影响智能体间的协调。我们将SMAC-Talk作为一个开放基准发布，以支持研究社区在协作多智能体环境中开发和评估LLM智能体。

## Abstract
As LLMs become more widely deployed, they are increasingly expected to work alongside other AI agents rather than operating in isolation. Effective coordination in these settings requires agents to communicate, share information and make decisions under uncertainty. We introduce SMAC-Talk, a natural language extension of the StarCraft Multi-Agent Challenge for evaluating LLM-based agents in cooperative multi-agent environments. The environment has several key features such as decentralized control, partial observability and long-horizon decision making. SMAC-Talk includes a natural language communication channel which is used to probe agent coordination and trust. We use this communication channel to construct different evaluation scenarios, including settings with an embedded deceptive communicator that tries to disrupt and deceive allies through communication alone. We provide three agents for benchmarking using 4 models from the Qwen3.5 family and study how reasoning structure, memory and model scale affect coordination between agents. We release SMAC-Talk as an open benchmark to support the research community in developing and evaluating LLM agents in cooperative multi-agent settings.

- **整体含义**：本文提出SMAC-Talk，通过将SMACv2的观测和动作转换为自然语言，并引入智能体间的自然语言通信通道，填补了这一空白。该基准不仅支持LLM智能体的协调评估，还通过嵌入欺骗性通信者的场景，探索LLM在对抗性语言下的信任与鲁棒性，这对于真实世界多智能体系统的安全部署具有重要意义。

## 2. 论文提出的方法论

### 核心思想
- 在SMACv2（星际争霸多智能体挑战）基础上，将每个智能体的局部数值观测转化为自然语言描述，智能体以自然语言选择动作，并允许通过通信通道交换自由文本消息。环境保留了SMACv2的核心特性：分散执行、部分可观测、共享奖励。

### 关键技术细节
- **观测到文本适配器**：将SMACv2的数值观测（自身、可见友军、可见敌军）转换为结构化自然语言。例如：“类型：Marine，血量：100%，位置：(14.9, 14.9)，可见盟友：Ally 1 (Marauder, 100%, 距离1.1)”。若通信启用，还会包含盟友前一时刻发送的消息。
- **文本到动作适配器**：解析LLM输出的自然语言动作（如“move north”、“attack Enemy 3”），映射为离散游戏动作。无效或无法解析的动作被替换为stop动作，并记录错误。
- **自然语言通信层**：每个智能体每步可提交一条消息，在下一时刻广播给所有能感知到它的友军。消息不经过过滤或修改，保持部分可观测性。
- **评估场景**：
  - 队伍规模：5v5和10v10（友军对敌军各5或10个单位）。
  - 通信类型：无通信(no_comm)、自由通信(comm)、已知欺骗性通信者(KDC)、未知欺骗性通信者(UDC)。KDC中智能体被告知可能存在欺骗者，UDC中未被告知。
  - 欺骗性通信者：受规则约束的智能体，不参与战斗，仅通过移动和通信干扰盟友（如谎报敌情、建议较差的战术）。
- **智能体设计**：
  - Zero-Shot Agent：极简指令，无推理解释，非思考模式。
  - ReAct Agent：结构化链式思考，包含游戏启发式（聚焦火力、定位、通信规则），维护过去2步历史，非思考模式。
  - Reasoning Agent：使用Qwen3.5内部思考模式（1024 token预算），自由推理，维护过去2步观测历史。
  - 欺骗性通信者：根据推理模式有两种提示变体。

## 3. 实验设计

### 数据集/场景
- 8个评估场景（见表1）：5v5_no_comm, 5v5_comm, 5v5_KDC, 5v5_UDC, 10v10_no_comm, 10v10_comm, 10v10_KDC, 10v10_UDC。
- 使用SMACv2的随机单位组成（Marine概率0.5，Marauder概率0.5）和位置生成（surround/reflect）。
- 敌军由StarCraft II内置AI控制，难度为“Very Easy”。

### Benchmark
- 这是一个新基准，本文自身提供了基线结果。没有与其他现有多智能体LLM方法直接比较，因为它是首个此类具身协作基准。

### 对比的方法
- 三种合作智能体（Zero-Shot、ReAct、Reasoning）与四种Qwen3.5模型规模（4B、9B、27B、122B-A10B）的交叉组合，共3×4=12种配置，在每个场景上评估。

### 评价指标
- **胜率**：全歼敌军比例。
- **平均奖励**：SMACv2奖励信号（基于伤害、消灭、胜利），提供更细粒度评估。
- **动作错误率**：无效或格式错误动作的比例。

## 4. 资源与算力

- **GPU型号**：Nvidia H100。
- **模型服务**：使用vLLM。
- **具体配置**：
  - 4B模型：单张H100
  - 9B和27B模型：2张H100
  - 122B-A10B模型：4张H100
- **推理时间**：Reasoning、ReAct、Zero-Shot智能体完成所有场景分别约需24、18、6小时（对于一个模型规模）。

## 5. 实验数量与充分性

- **实验数量**：每个场景运行100个episode，共8个场景，4种模型规模，3种智能体类型，共计8×4×3=96个实验条件（每个条件100episode是足够的统计量）。另外还有欺骗性通信者的两种变体（KDC/UDC）也包含在内。
- **充分性**：
  - 覆盖了两种队伍规模、四种通信模式，消融了通信和欺骗性情境的影响。
  - 对比了不同推理结构、记忆、模型规模。
  - 报告了胜率、奖励、错误率，从多角度衡量性能。
  - 缺点：仅使用Qwen3.5系列，未扩展到其他模型族；仅使用Terran单位；仅固定敌军难度；ReAct通信崩溃原因未深入探因。总体而言实验设计较为充分，但泛化性有限。

## 6. 论文的主要结论与发现

- **推理方式**：内部链式思考（Reasoning Agent）始终优于Zero-Shot和ReAct，无论模型规模。结构化外部推理（ReAct）在通信场景下性能崩溃，内部思考则更具鲁棒性。
- **模型规模**：4B模型表现不足以可靠协调，9B是有效下限，27B和122B-A10B之间性能增长趋于饱和。较大模型能更好利用通信并抵抗欺骗。
- **通信效果**：通信对Zero-Shot和Reasoning Agent有适度或变化的影响，但ReAct Agent在通信场景下性能显著下降（所有规模）。原因不明，可能与提示结构不兼容有关。
- **抵抗欺骗**：在已知欺骗者（KDC）场景下，大模型能有效折扣不可靠通信（如122B Reasoning在5v5 KDC胜率41% vs UDC 10%）。在未知欺骗者（UDC）场景下，大模型相对更脆弱，但仍优于小模型。
- **架构影响**：提示架构选择在多智能体场景中影响巨大，单智能体的成功设计可能无法直接迁移。

## 7. 优点

- **首次将SMACv2转化为自然语言基准**，弥合了语言多智能体模拟器与传统MARL基准之间的差距。
- **引入自然语言通信通道**，并设计欺骗性通信者场景，可系统研究LLM的信任、鲁棒性和协调能力，这在现有基准中缺乏。
- **开源环境**，支持多种推理后端（vLLM、Llama.cpp、Cerebras、OpenAI兼容API），便于研究人员复现和扩展。
- **实验设计较全面**：覆盖不同模型规模、多种推理结构、不同通信条件，提供多指标（胜率、奖励、错误率）评估。
- **欺骗性通信者规则清晰**，确保失败源于协调和信任问题而非实现漏洞。

## 8. 不足与局限

- **计算成本高**：即使使用小模型（9B）也需要较长时间，对资源有限的研究者不够友好。
- **模型泛化性有限**：仅评估Qwen3.5系列，其他模型家族（如GPT、LLaMA）的表现未知。
- **环境限制**：仅使用Terran单位、固定敌军难度（Very Easy），未探索种族多样性、难度梯度、更大队列规模。
- **智能体设计**：ReAct的通信崩溃原因未被充分解释；Zero-Shot和ReAct的提示未进行元优化（heuristic tuning），可能并非该场景下的最佳设计。
- **动作错误率指标问题**：无效动作被替换为stop，可能惩罚格式不一致但推理正确的模型，混淆评估。
- **未探索推理令牌预算的影响**：Reasoning Agent固定1024 token，不同预算对协调的影响未知。
- **可能未考虑长期记忆**：虽然维护过去2步历史，但更长的记忆或更复杂的记忆机制未被评估。
- **欺骗性通信者能力有限**：仅通过通信干扰，未测试更复杂的欺骗策略（如反向信号、动态调整）。
