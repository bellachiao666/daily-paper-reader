## 摘要
通信协议设计是基于大型语言模型的多智能体系统中的核心挑战。现有的单通道方法面临固有的通信三难困境：基于文本的方法可解释但冗长，而隐空间方法高效但不透明且仅限于单向工作流。受多通道通信理论启发，我们提出HyLaT，一种混合隐式-文本通信协议，通过隐通道传输精细的认知信号以实现高效，同时用自然语言表达简洁的关键信号以保持可解释性和精确性。我们引入了一个两阶段训练框架，结合单智能体混合生成学习与多智能体交互协同训练，使智能体能够在多轮交互中生成和解释混合消息。实验表明，HyLaT在显著降低通信开销的同时保持有竞争力的任务性能，并在不同设置下展现出强大的泛化能力和鲁棒性。

## Abstract
Communication protocol design is a central challenge in large language model-based multi-agent systems. Existing single-channel approaches face an inherent communication trilemma: text-based methods are interpretable but verbose, while latent-space methods are efficient but opaque and limited to unidirectional workflows. Inspired by multi-channel communication theory, we propose HyLaT, a hybrid latent-text communication protocol that transmits elaborate cognitive signals through a latent channel for efficiency, while expressing concise critical signals in natural language to preserve interpretability and precision. We introduce a two-stage training framework combining single-agent hybrid generation learning and multi-agent interactive co-training, enabling agents to generate and interpret hybrid messages across multiple rounds of interaction. Experiments demonstrate that HyLaT reduces communication overhead significantly while maintaining competitive task performance, with strong generalization and robustness across diverse settings.

## 1. 核心问题与整体含义（研究动机和背景）

- **核心问题**：基于LLM的多智能体系统（MAS）中，通信协议设计面临“三元困境”（communication trilemma）：文本方法（如自然语言）可解释但冗长、效率低；隐空间方法（如Cipher、LatentMAS）高效但完全不透明，且仅支持单向/序列化工作流，无法支持多轮交互。现有单通道协议无法同时优化效率（efficiency）、可移植性（portability）和多功能性（versatility）。
- **研究动机**：受多通道通信理论（Shannon, 1948; Monge & Contractor, 2003）启发，认为不同类型的消息（如详细推理 vs. 最终决策）应通过不同信道传输。智能体消息实际上包含两种功能不同的信息：**详尽的认知信号**（解释、推理过程）适合紧凑的隐空间表示；**简洁的关键信号**（答案、承诺）必须保持人类可读。
- **整体含义**：通过设计双通道混合协议，有望在保持可解释性的同时大幅提升通信效率，解决三元困境，并支持多轮、多智能体交互。

## 2. 方法论：核心思想、关键技术细节、公式或算法流程

### 核心思想
提出 **HyLaT (Hybrid Latent-Text communication protocol)**，智能体同时输出两部分：
- **隐通道（Latent Channel）**：传输 k 个连续向量（隐表示），承载详细推理/认知信号。
- **文本通道（Text Channel）**：用自然语言传输简洁的关键信号（如最终答案、结论）。

双通道设计使智能体既能高效交换密集信息，又能保持对外部观察者（用户）的透明性。

### 关键技术细节

1. **双通道响应格式**（公式2）：
   ```
   o_i^{(t)} = [⟨bot⟩ z_1, ..., z_k ⟨eot⟩]  ∥  [y_1, ..., y_m]
   ```
   其中 `⟨bot⟩`、`⟨eot⟩` 为可学习特殊标记，`z` 为隐向量，`y` 为文本标记。

2. **两阶段训练框架**：

   **Stage 1：单智能体混合生成学习**  
   - 输入纯文本 `x`，目标响应包含文本解释 `e` 和关键输出 `y`。  
   - 每次训练执行两次前向传播：  
     - **混合前向**：模型自回归生成 `k` 个隐向量 `Z`，然后生成 `y`，仅对 `y` 计算语言模型损失 `L_hybrid`（公式3）。  
     - **纯文本前向**：模型生成完整 `[e, y]`，教师强制计算 `L_text`（公式4）。  
     - **跨通道对齐损失**：在 `e` 结束、`y` 开始前的共享位置，将纯文本分支的隐状态 `h_T^l` 与混合分支的隐状态 `h_S^l` 对齐，使用 stop-gradient（公式5）：
       ```
       L_align = 1/L Σ_l || sg[h_T^l] - h_S^l ||²
       ```
     - 总损失（公式6）：`L_stage1 = α L_hybrid + β L_text + γ L_align`。

   **Stage 2：多智能体交互协同训练**  
   - 构建多轮、多智能体对话数据（Refinement 和 Decomposition 两种类型）。  
   - 每个智能体基于历史上下文和上一轮其他智能体的输出（包含隐向量和文本）生成当前轮输出。  
   - 对每个智能体计算 `L_stage1` 对应损失，平均所有智能体损失（公式7）：  
     ```
     L_stage2 = 1/N Σ_i (α L_hybrid^(i) + β L_text^(i) + γ L_align^(i))
     ```
   - 通过联合训练，智能体学会解读同伴的隐向量并生成混合响应。

### 算法流程（文字描述）
1. 初始化共享骨干模型（如 Llama-3.2-1B-Instruct），LoRA 微调。
2. Stage 1：在单智能体数据集（含 `e` + `y`）上训练，优化 `L_stage1`，使模型学会自行生成隐向量并基于隐向量输出文本答案。
3. Stage 2：使用人工模拟的多智能体对话数据（多轮），每个智能体观察其他智能体的隐向量和文本，输出自己的混合响应，优化平均损失。训练后即可在测试时实现多轮混合通信。

## 6. 主要结论与发现

1. **训练无关方法无法平衡效率与性能**：文本方法效率低，隐空间方法性能大幅下降。
2. **HyLaT实现了效率和性能的平衡**：
   - 通信效率：比最高效的文本方法提升10.6× token、6.8×时间；比隐空间方法提升3.6× token、2.5×时间；接近单智能体基线。
   - 任务性能：在多数数据集上达到最优或次优，尤其 SocialIQA 上显著优于所有对比方法。
   - 优于 TextFullT（性能相当但开销巨大）和 LatentFullT（性能略低）。
3. **双通道设计至关重要**：去掉任一通道都会导致效率下降或性能损失。
4. **两阶段训练必须**：缺少 Stage 1 导致隐向量不稳定；缺少 Stage 2 导致智能体无法处理同伴的隐向量。
5. **兼容性与鲁棒性**：
   - 可与纯文本智能体混用，降低整体开销并提升性能。
   - 对隐向量噪声（σ≤0.5）稳定，因关键信息在文本通道。
6. **可扩展性**：隐向量数量/智能体数量/轮数增加时，HyLaT token增长远小于其他方法；增加训练数据可缓解性能下降。

## 7. 优点

- **创新性**：首次在多智能体通信中引入双通道混合协议，思想干净、直接解决三难困境。

## 8. 不足与局限

- **模型规模受限**：仅测试了1B/3B小模型，更大模型（如7B/70B）上的效果未知。
- **部署限制**：自定义生成流程（隐向量自回归+文本）与 vLLM 等推理框架不兼容，无法直接用于大规模长时序场景。

- **训练-评估配置差异**：主要实验使用 N=2, T=2 训练，但评估中扩展到更大规模时性能下降，需额外数据微调才能缓解。

- **基线适配问题**：LatentMAS-V 改编后性能可能被低估，但作者已说明。
