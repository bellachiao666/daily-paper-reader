## 摘要
基于大型语言模型（LLM）构建的多智能体系统需要许多难以事先确定的协调选择：调用哪种技能协议、哪个智能体角色应执行子任务、每个角色绑定哪个模型、角色应如何交互、何时使用检索或验证、以及何时完全省略步骤。这些选择与任务机制和操作约束相互作用，因此静态管道和一次性的模型比较只能提供设计空间的有限视角。本文介绍了AgensFlow，这是一个开源框架，将多智能体协调视为部分可观测条件下的在线策略学习问题。该框架使协调决策可观察并可从重复轨迹中学习，而不是将技能、角色、模型、拓扑和评估选择视为固定的管道设计。AgensFlow在两个数据集上进行了评估：分布式系统事件任务和安全咨询任务。评估显示了三个主要结果：在协调密集型类别中，学习到的路由比固定管道基线达到了更高质量的工作点；skip:X将拓扑压缩作为基础架构的一个有意义的部分；而热启动的策略图可以减少探索成本，同时保持平稳质量。总体而言，结果表明，相比于静态布线，可学习、可审计的路由可以改善协调密集型多智能体工作流。

## Abstract
Multi-agent systems built on large language models (LLMs) require many coordination choices that are difficult to fix a priori: which skill protocol to invoke, which agent role should perform a subtask, which model to bind to each role, how roles should interact, when to use retrieval or verification, and when to omit a step entirely. These choices interact with task regime and operational constraints, so static pipelines and one-off model comparisons provide only a limited view of the design space. This paper introduces AgensFlow, an open-source framework that treats multi-agent coordination as an online policy-learning problem under partial observability. The framework makes coordination decisions observable and learnable from repeated trajectories, rather than treating skill, role, model, topology, and evaluation choices as fixed pipeline design.   AgensFlow is evaluated on two corpora: distributed-systems incident tasks and security-advisory tasks. The evaluation shows three main results: learned routing reaches a higher-quality operating point than a fixed pipeline baseline on coordination-heavy classes; skip:X isolates topology compression as a meaningful part of the substrate; and warm-started policy graphs can reduce exploration cost while preserving plateau quality. Overall, the results support that learned, auditable routing can improve coordination-heavy multi-agent workflows over static wiring.

- **整体含义**：提出将多智能体协调视为一个在线策略学习问题，通过可审计的策略图在重复轨迹中学习协调决策，从而替代手工编写的固定工作流。核心贡献在于让“跳过步骤”（skip）成为一等公民，并提供跨领域转移学习的能力。

## 2. 论文提出的方法论：核心思想、关键技术细节

### 核心思想
- 将多智能体编排建模为**部分可观测的序贯决策过程**，学习一个**状态抽象化的策略图**。状态不是原始提示，而是**折叠后的任务签名（folded signature）**，由任务类别标签、二进制握手指令掩码、以及连续信念估计（正确性、不确定性、矛盾风险、证据充分性）组成。
- 动作空间包括：`invoke(k, m)`（调用技能协议k绑定模型m）、`skip:X`（跳过仍计划中的可选单元X）、以及隐式终止（由评估器或预算耗尽触发）。拓扑本身就是动作空间的一部分，因此是学习得到的。
- 使用**可靠性感知的UCB1变体**进行动作选择，融合平均奖励、探索系数（随访问次数退火）、以及失败惩罚项。
- 奖励函数：`r(t) = w_q * Q(t) - w_c * C(t) - w_ρ * ρ(t)`，其中Q(t)是RelativeJudge质量分数（同任务轨迹相对排名），C(t)是归一化token成本，ρ(t)是重试/失败惩罚。

### 关键技术细节
1. **状态折叠（folded signature）**：`s_t = ϕ(x_t, h_t, b_t)` 包含：
   - 规则检测器输出的类别标签（straightforward, evidence_heavy, ambiguous, contradictory, high_risk, exploratory）
   - 7位二进制握手指令掩码（goal, subproblem, evidence, critique, verification, draft_answer, merged_answer是否已填充）
   - 4个连续信念估计（经桶化压缩）：正确性、不确定性、矛盾风险、证据充分性
2. **技能变体池**：9个（技能协议×模型）求解器变体，加规划器、记忆、两种网络搜索、两种验证器、评估器。
3. **策略图**：每个(s,a)对存储访问次数、平均奖励、奖励方差、token统计、失败次数。
4. **选择规则**：`score(s,a) = mean_reward(s,a) + c_s * sqrt(log(N_s+1)/N_{s,a}) - λ*failure_rate(s,a)`，其中`c_s`随访问次数退火。
5. **训练过程**：每任务经历：预检→特征计算→签名折叠→循环选择动作→执行→更新信念→重新计算签名→终止→RelativeJudge打分→混合奖励→备份到策略图。

## 6. 论文的主要结论与发现

1. **学习路由优于固定流水线**：在协调密集型类别（C3跨文档多供应商推理、C7缓解正确性、C8跨供应商对）上，主实验的跨家族审计质量比固定基线提升显著（C3:+0.181, C7:+0.131, C8:+0.156），而在程序性类别上持平或略有下降。
2. **`skip:X` 是有效的拓扑压缩形式**：无skip消融的token成本比基线高94%（仅8轮学习），而主实验仅增加15%但质量提升+0.068。表明跳过动作能学习到成本效益更优的轨迹。
3. **热启动减少探索成本并保持质量**：热启动在8轮中平均token成本低约10%（平稳期），跨家族审计质量与冷启动相当（平稳期+0.012，全运行均值+0.003），且在C5和C7上有小幅度提升。
4. **奖励信号鲁棒性是协调问题的一部分**：单评判显示热启动有较大质量优势，但跨家族审计缩小了这种差距（从+0.052降到+0.012）。说明单评判可能存在家庭偏见，奖励审计应集成到系统设计中。
5. **策略适应任务类别**：在7/8个类别上，系统从默认的`solver_cot_haiku`移开，选择了证据模板、简洁模板或保留默认（仅在C7高风险类别保留默认），表明策略根据任务结构和风险进行了自适应调整。
